# 空安全

> 上一章我们实现了文件持久化，但代码中有很多 `null` 检查。本章学习 Dart 的空安全机制，让代码更安全、更优雅。

Dart 2.12 起强制启用空安全（Sound Null Safety）。变量默认不可为 null，必须显式声明可空类型。

## 核心原则

1. **默认不可空** — 除非你主动声明，否则类型不接受 null
2. **完全安全** — 编译器保证可空变量在使用前已检查
3. **渐进迁移** — 可以逐步将代码迁移到空安全

## 可空类型

用 `?` 后缀表示可空：

```dart
String name = 'Dart';    // 不可空，永远不为 null
String? nickname = null;  // 可空，可以为 null

// 不可空变量必须初始化
// String title;          // ❌ 编译错误
String title = '';        // ✅ 有初始值
```

### 基本类型的可空版本

```dart
int? age;
double? price;
bool? isValid;
List<String>? items;
Map<String, dynamic>? data;
```

## 空检查操作符

### ?. — 安全访问

```dart
String? name;
print(name?.length);   // null（name 为 null 时不调用 .length）
print(name?.toUpperCase());  // null

// 链式安全访问
String? city;
print(city?.length?.isEven);  // null
```

### ?? — 空值合并

```dart
String? name;
var display = name ?? '匿名';   // name 为 null 时使用 '匿名'
print(display);  // 匿名

// 链式使用
var result = a ?? b ?? c ?? '默认';
```

### ??= — 空值赋值

```dart
int? count;
count ??= 0;    // count 为 null，赋值为 0
count ??= 99;   // count 不为 null，不赋值
print(count);   // 0
```

### ! — 非空断言

告诉编译器"我确定这不是 null"，如果实际为 null 会抛出异常：

```dart
String? name = 'Dart';
print(name!.length);  // 4

String? empty;
// print(empty!.length);  // ❌ 运行时抛出 NullPointerException
```

> 谨慎使用 `!`，它绕过了编译器的空安全检查。只在你能 100% 确定不为 null 时使用。

## 类型提升

编译器会自动将可空类型"提升"为不可空类型：

```dart
void printLength(String? text) {
  // 这里 text 是 String?，不能直接用 .length
  if (text != null) {
    // 类型提升：text 自动变为 String
    print(text.length);  // ✅ 安全
  }
}

// is 检查也会提升
void process(Object value) {
  if (value is String) {
    print(value.length);  // ✅ 自动提升为 String
  }
}
```

### 局部变量提升

```dart
void check(String? name) {
  if (name == null) return;  // 早返回

  // name 已提升为 String
  print(name.length);  // ✅
  print(name.toUpperCase());  // ✅
}
```

## late 关键字

`late` 表示变量稍后初始化，但使用时保证不为 null：

```dart
class Config {
  late String apiKey;

  void init(String key) {
    apiKey = key;
  }

  void use() {
    print(apiKey.length);  // 如果 init 未调用，运行时抛出错误
  }
}
```

### late 的典型场景

```dart
// 1. 依赖构造函数参数计算
class Rectangle {
  late final double area;

  Rectangle(double width, double height) {
    area = width * height;
  }
}

// 2. 延迟初始化（避免循环依赖）
class Controller {
  late final Service service;

  void setup() {
    service = Service();
  }
}

// 3. Flutter 中的 TextEditingController
class _MyWidgetState extends State<MyWidget> {
  late TextEditingController controller;

  @override
  void initState() {
    super.initState();
    controller = TextEditingController();
  }
}
```

### late vs nullable

| 特性 | late | nullable |
|------|------|----------|
| 初始值 | 无（使用前必须赋值） | null |
| 使用时 | 直接用，无需空检查 | 需要 `?.` 或 `!` |
| 误用后果 | 运行时 LateInitializationError | 运行时 NullPointerException |
| 适用场景 | 确定会初始化 | 可能永远为 null |

## 函数参数的空安全

```dart
// 不可空参数：必须传值
void greet(String name) {
  print('Hello, $name');
}

// 可空参数：可以传 null
void greetOptional(String? name) {
  print('Hello, ${name ?? "World"}');
}

// 默认值
void greetDefault(String name = 'World') {
  print('Hello, $name');
}

// required 命名参数
void createUser({required String name, int? age}) {
  print('$name, $age');
}
```

## 集合的空安全

```dart
// List<String?> — 元素可以为 null
List<String?> names = ['Dart', null, 'Flutter'];

// List<String>? — 列表本身可以为 null
List<String>? tags = null;

// 展开运算符的空安全
List<int>? a = [1, 2, 3];
List<int>? b = null;
var combined = [...?a, ...?b];  // [1, 2, 3]
```

## 常见模式

### 早返回模式

```dart
String getDisplayName(User? user) {
  if (user == null) return '匿名';
  return user.name;
}
```

### 空值链式处理

```dart
// 多层可空属性
String? getCity(Person? person) {
  return person?.address?.city;
}
```

### 条件赋值

```dart
String? input;
var label = input?.toUpperCase() ?? 'N/A';
```

## 小结

| 语法 | 用途 | 安全性 |
|------|------|--------|
| `Type?` | 声明可空类型 | 编译期检查 |
| `?.` | 安全访问属性/方法 | 安全 |
| `??` | 空值合并 | 安全 |
| `??=` | 空值赋值 | 安全 |
| `!` | 非空断言 | 不安全，运行时可能崩溃 |
| `late` | 延迟初始化 | 不安全，使用前未初始化会崩溃 |

## 踩坑提示

::: warning 常见错误
1. **滥用 `!`** — 每写一个 `!` 都要问自己：我能 100% 保证不为 null 吗？优先用 `?.` 和 `??`
2. **`late` 不是可选** — `late` 意味着"我保证稍后会初始化"，不是"可能不初始化"
3. **可空类型的集合** — `List<String?>` 和 `List<String>?` 是不同的类型
4. **类型提升有边界** — 闭包中捕获的变量不会被提升，因为闭包可能在其他地方执行
5. **`??` 链式使用** — `a ?? b ?? c ?? 'default'` 从左到右取第一个非 null 值
:::

## 实战：空安全改造 Todo 应用

用空安全最佳实践改造之前的代码：

```dart
class Todo {
  final int id;
  final String content;
  bool completed;
  final DateTime createdAt;

  Todo({
    required this.id,
    required this.content,
    this.completed = false,
    DateTime? createdAt,
  }) : createdAt = createdAt ?? DateTime.now();

  // 从 JSON 构造，字段可能缺失
  factory Todo.fromJson(Map<String, dynamic> json) {
    return Todo(
      id: json['id'] as int,
      content: json['content'] as String,
      completed: json['completed'] as bool? ?? false,
      createdAt: json['createdAt'] != null
        ? DateTime.parse(json['createdAt'] as String)
        : null,
    );
  }

  Map<String, dynamic> toJson() => {
    'id': id,
    'content': content,
    'completed': completed,
    'createdAt': createdAt.toIso8601String(),
  };
}

// 使用 ?. 和 ?? 安全处理
void main() {
  Todo? todo = repository.findById(1);

  // 安全访问
  print(todo?.content ?? '未找到');

  // 早返回模式
  if (todo == null) {
    print('Todo 不存在');
    return;
  }
  // 这里 todo 已提升为非空 Todo
  print(todo.content.length);
}
```
