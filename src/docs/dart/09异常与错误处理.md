# 异常与错误处理

> 上一章我们用空安全改造了 Todo 应用。但即使类型安全，运行时仍可能出错（文件损坏、网络中断等）。本章学习异常处理，让应用更健壮。

Dart 的异常处理机制与 Java/C# 类似，但更简洁。

## Exception vs Error

| 类型 | 性质 | 是否应该捕获 | 典型例子 |
|------|------|------------|---------|
| Exception | 可预期的异常 | 是 | FormatException, IOException |
| Error | 不可恢复的错误 | 否 | StackOverflowError, OutOfMemoryError |

- **Exception**：程序逻辑可以处理的异常情况
- **Error**：程序 bug 或系统级问题，不应该捕获

## try / on / catch / finally

### 基本用法

```dart
try {
  var result = 10 ~/ 0;
} on IntegerDivisionByZeroException {
  print('不能除以零');
}
```

### catch 捕获异常对象

```dart
try {
  var result = 10 ~/ 0;
} catch (e) {
  print('出错了：$e');  // 出错了：IntegerDivisionByZeroException
}
```

### 获取堆栈跟踪

```dart
try {
  var result = 10 ~/ 0;
} catch (e, stackTrace) {
  print('异常：$e');
  print('堆栈：$stackTrace');
}
```

### on + catch 组合

```dart
try {
  riskyOperation();
} on FormatException catch (e) {
  print('格式错误：$e');
} on NetworkException catch (e) {
  print('网络错误：$e');
} catch (e) {
  print('未知错误：$e');
}
```

### finally

```dart
try {
  var data = await fetchData();
  processData(data);
} catch (e) {
  print('处理失败：$e');
} finally {
  // 无论成功失败都执行
  hideLoading();
}
```

## throw — 抛出异常

### 抛出标准异常

```dart
void setAge(int age) {
  if (age < 0) {
    throw ArgumentError('年龄不能为负数：$age');
  }
  if (age > 150) {
    throw RangeError('年龄不合理：$age');
  }
}
```

### 抛出任意对象

Dart 可以抛出任何非 null 对象：

```dart
throw '出错了！';           // 字符串
throw 42;                   // 数字
throw Exception('失败');    // Exception 对象
```

> 最佳实践：总是抛出 Exception 或其子类，不要抛出字符串或数字。

## 自定义异常

```dart
class AppException implements Exception {
  final String message;
  final int? code;

  AppException(this.message, {this.code});

  @override
  String toString() => code != null
    ? 'AppException($code): $message'
    : 'AppException: $message';
}

// 使用
class NetworkException extends AppException {
  final String url;

  NetworkException(String message, this.url, {int? code})
    : super(message, code: code);
}

// 抛出
throw NetworkException('连接超时', 'https://api.example.com', code: 408);

// 捕获
try {
  await apiCall();
} on NetworkException catch (e) {
  print('网络错误：${e.message}，URL：${e.url}');
} on AppException catch (e) {
  print('应用错误：$e');
}
```

## 常见内置异常

| 异常 | 触发场景 |
|------|---------|
| `FormatException` | 字符串解析失败 |
| `ArgumentError` | 参数不合法 |
| `RangeError` | 索引越界 |
| `StateError` | 对象状态不正确 |
| `TypeError` | 类型错误（空安全相关） |
| `IntegerDivisionByZeroException` | 整数除以零 |
| `TimeoutException` | 异步操作超时 |
| `IOException` | 文件/网络 I/O 错误 |

## 异步代码的错误处理

### try/catch + async

```dart
Future<void> loadData() async {
  try {
    var data = await fetchUser();
    print(data);
  } on TimeoutException {
    print('请求超时，请重试');
  } on FormatException catch (e) {
    print('数据格式错误：$e');
  } catch (e) {
    print('未知错误：$e');
  }
}
```

### Future 的 catchError

```dart
fetchUser()
  .then((data) => processData(data))
  .catchError((error) {
    if (error is TimeoutException) {
      print('超时');
    } else {
      print('错误：$error');
    }
  });
```

### Stream 的错误处理

```dart
stream.listen(
  (data) => print('数据：$data'),
  onError: (error) {
    print('流错误：$error');
  },
  onDone: () {
    print('流结束');
  },
);
```

## rethrow

在 catch 中重新抛出异常，让上层处理：

```dart
Future<void> loadWithRetry() async {
  try {
    var data = await fetchData();
    process(data);
  } catch (e) {
    logError(e);  // 记录日志
    rethrow;      // 继续向上抛出
  }
}
```

## 最佳实践

### 1. 用 on 区分异常类型

```dart
// ❌ 不推荐：笼统捕获
try {
  await operation();
} catch (e) {
  print('出错了');
}

// ✅ 推荐：区分处理
try {
  await operation();
} on NetworkException {
  showRetryDialog();
} on AuthException {
  redirectToLogin();
} catch (e) {
  reportToSentry(e);
}
```

### 2. 不要捕获 Error

```dart
// ❌ 不要捕获 Error
try {
  list[100];
} on RangeError {
  // 这掩盖了 bug
}

// ✅ 修复代码本身
if (index < list.length) {
  return list[index];
}
```

### 3. 资源清理放 finally

```dart
File? file;
try {
  file = File('data.txt');
  var content = await file.readAsString();
  return parse(content);
} catch (e) {
  logError(e);
  rethrow;
} finally {
  // 确保资源释放
  await file?.close();
}
```

### 4. 自定义异常继承体系

```dart
abstract class AppException implements Exception {
  final String message;
  AppException(this.message);
}

class NetworkException extends AppException {
  final int? statusCode;
  NetworkException(String message, {this.statusCode}) : super(message);
}

class AuthException extends AppException {
  AuthException(String message) : super(message);
}

class ValidationException extends AppException {
  final Map<String, String> errors;
  ValidationException(String message, this.errors) : super(message);
}
```

## 小结

- 用 `on` 区分异常类型，`catch` 获取异常对象
- `finally` 用于资源清理，无论成功失败都执行
- 自定义异常继承 `Exception`，建立异常体系
- 不要捕获 `Error`，那是 bug，应该修复
- 异步代码优先用 `async/await` + `try/catch`

## 踩坑提示

::: warning 常见错误
1. **空 catch 吞掉异常** — `catch (e) {}` 让错误无声消失，至少要 `log`
2. **捕获 Error** — `on Error catch` 掩盖了程序 bug，应该修复代码而不是捕获
3. **finally 中的异常** — finally 里抛出异常会覆盖 try 中的原始异常
4. **rethrow vs throw** — `rethrow` 保留原始堆栈，`throw e` 会丢失，优先用 `rethrow`
5. **异步异常不会冒泡** — `try { asyncFunc(); } catch` 捕获不到，必须 `await`
:::

## 实战：健壮的错误处理

为 Todo 应用的文件操作添加完善的错误处理：

```dart
// 自定义异常
class TodoException implements Exception {
  final String message;
  final String? detail;
  TodoException(this.message, {this.detail});

  @override
  String toString() => detail != null
    ? 'TodoException: $message ($detail)'
    : 'TodoException: $message';
}

class TodoRepository {
  // ...

  Future<void> load() async {
    try {
      var file = File(_filePath);
      if (!await file.exists()) return;

      var content = await file.readAsString();
      var json = jsonDecode(content) as List;
      _todos.clear();
      for (var item in json) {
        _todos.add(Todo.fromJson(item as Map<String, dynamic>));
      }
    } on FormatException catch (e) {
      throw TodoException('数据文件格式错误', detail: e.message);
    } on FileSystemException catch (e) {
      throw TodoException('文件读取失败', detail: e.message);
    } on TypeError catch (e) {
      throw TodoException('数据字段类型不匹配', detail: e.toString());
    }
  }

  Future<void> safeSave() async {
    try {
      await save();
    } catch (e) {
      // 保存失败不影响主流程，但记录日志
      print('警告：保存失败 - $e');
    }
  }
}
```
