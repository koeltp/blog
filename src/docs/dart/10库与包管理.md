# 库与包管理

> 上一章我们给 Todo 应用加了健壮的错误处理。现在代码越来越长了，本章学习模块化开发，把单文件项目拆分为多文件结构。

Dart 通过库（Library）和包（Package）实现代码模块化和复用。

## 库（Library）

### import 导入

```dart
// 导入核心库
import 'dart:math';
import 'dart:io';
import 'dart:convert';

// 使用
print(pi);                          // 3.14159...
print(Random().nextInt(100));       // 随机数
print(json.encode({'key': 'value'})); // JSON 编码
```

### 导入自定义文件

```dart
// 相对路径导入
import 'lib/utils.dart';
import 'lib/models/user.dart';

// 包内导入
import 'package:my_app/utils.dart';
```

### as — 命名空间

当两个库有同名类/函数时，用 `as` 消除歧义：

```dart
import 'dart:math' as math;
import 'dart:ui' as ui;

print(math.pi);
print(ui.Color(0xFF000000));
```

### show / hide — 选择性导入

```dart
// 只导入需要的
import 'dart:math' show pi, Random;

// 排除不需要的
import 'dart:math' hide max;
```

### export — 导出

组合多个文件为一个库的公开 API：

```dart
// lib/my_package.dart
export 'src/service_a.dart';
export 'src/service_b.dart';
export 'src/models.dart' show User, Product;
```

使用者只需导入一个文件：

```dart
import 'package:my_package/my_package.dart';
```

## Dart 核心库

| 库 | 用途 |
|------|------|
| `dart:core` | 内置类型（int, String, List, Map...），自动导入 |
| `dart:math` | 数学函数、随机数 |
| `dart:io` | 文件、网络、进程（仅 VM，不支持 Web） |
| `dart:convert` | JSON、UTF-8 编解码 |
| `dart:async` | Future、Stream、Completer |
| `dart:collection` | 高级集合类（Queue, LinkedHashMap...） |
| `dart:typed_data` | 高效二进制数据 |

### dart:convert 常用操作

```dart
import 'dart:convert';

// JSON 编码
var json = jsonEncode({'name': 'Dart', 'version': 3});
print(json);  // {"name":"Dart","version":3}

// JSON 解码
var data = jsonDecode('{"name":"Dart","version":3}');
print(data['name']);  // Dart

// 类型安全的 JSON 解析
Map<String, dynamic> parsed = jsonDecode(json);
String name = parsed['name'] as String;
int version = parsed['version'] as int;

// UTF-8 编解码
var bytes = utf8.encode('你好');
var text = utf8.decode(bytes);
```

## 包（Package）

### pub.dev

Dart 的官方包仓库：[pub.dev](https://pub.dev/)

常用包：

| 包 | 用途 |
|------|------|
| `http` | HTTP 客户端 |
| `path` | 跨平台路径操作 |
| `intl` | 国际化与格式化 |
| `dio` | 高级 HTTP 客户端 |
| `provider` | 状态管理（Flutter） |
| `shared_preferences` | 本地键值存储（Flutter） |

### pubspec.yaml

包配置文件，位于项目根目录：

```yaml
name: my_app
description: 我的应用
version: 1.0.0

environment:
  sdk: '>=3.0.0 <4.0.0'

dependencies:
  http: ^1.1.0
  path: ^1.8.0
  intl: ^0.19.0

dev_dependencies:
  test: ^1.24.0
  lints: ^3.0.0
```

### 版本约束语法

| 语法 | 含义 |
|------|------|
| `any` | 任意版本（危险） |
| `1.2.3` | 精确版本 |
| `^1.2.3` | >=1.2.3 <2.0.0（推荐） |
| `>=1.2.3 <2.0.0` | 范围约束 |

### 常用 pub 命令

```bash
# 获取依赖
dart pub get

# 升级依赖
dart pub upgrade

# 升级到最新主版本
dart pub upgrade --major-versions

# 发布包到 pub.dev
dart pub publish

# 查看依赖树
dart pub deps

# 运行代码生成器
dart run build_runner build
```

## 创建自己的包

### 包结构

```
my_package/
├── lib/
│   ├── my_package.dart      # 主入口
│   └── src/
│       ├── service_a.dart
│       └── service_b.dart
├── test/
│   └── my_package_test.dart
├── example/
│   └── main.dart
├── pubspec.yaml
├── README.md
├── CHANGELOG.md
└── LICENSE
```

### 创建步骤

```bash
# 创建包项目
dart create -t package my_package

# 或 Flutter 插件
flutter create --template=package my_package
```

### 编写包代码

```dart
// lib/src/calculator.dart
class Calculator {
  int add(int a, int b) => a + b;
  int subtract(int a, int b) => a - b;
}
```

```dart
// lib/my_package.dart
export 'src/calculator.dart';
```

### 编写测试

```dart
// test/calculator_test.dart
import 'package:test/test.dart';
import 'package:my_package/my_package.dart';

void main() {
  test('加法', () {
    var calc = Calculator();
    expect(calc.add(2, 3), equals(5));
  });

  test('减法', () {
    var calc = Calculator();
    expect(calc.subtract(5, 3), equals(2));
  });
}
```

运行测试：

```bash
dart test
```

## 条件导入

根据平台导入不同实现：

```dart
// 默认导入 VM 实现，Web 平台导入 web 实现
import 'storage_vm.dart'
  if (dart.library.html) 'storage_web.dart';

// storage_vm.dart
String getStoragePath() => './data';

// storage_web.dart
String getStoragePath() => '/tmp/data';
```

## 小结

| 概念 | 关键点 |
|------|--------|
| `import` | 导入库，支持 `as`/`show`/`hide` |
| `export` | 组合多个文件为公开 API |
| `pubspec.yaml` | 项目配置和依赖声明 |
| `pub.dev` | 官方包仓库，数万可用包 |
| `^版本` | 推荐的版本约束方式 |
| 条件导入 | 跨平台代码的关键技术 |

## 踩坑提示

::: warning 常见错误
1. **`package:` vs 相对路径** — 包内推荐用 `package:my_app/...`，避免相对路径层级混乱
2. **循环导入** — A 导入 B，B 导入 A，编译报错。用接口或事件解耦
3. **`pubspec.lock` 要提交** — 应用项目提交 lock 文件保证依赖一致性，包项目不提交
4. **`dart:io` 不支持 Web** — Web 项目用 `dart:html`，跨平台用条件导入
5. **`export` 不要过度** — 只导出公开 API，内部实现不要暴露
:::

## 实战：拆分为多文件项目

把 Todo 应用从单文件拆分为标准项目结构：

```
todo_app/
├── bin/
│   └── todo_app.dart        # 入口
├── lib/
│   ├── todo_app.dart        # 主库（导出所有公开 API）
│   ├── src/
│   │   ├── models/
│   │   │   └── todo.dart    # Todo 数据类
│   │   ├── repository.dart  # TodoRepository
│   │   ├── exceptions.dart  # 自定义异常
│   │   └── commands.dart    # 命令处理
│   └── todo_app.dart
├── test/
│   └── todo_test.dart
└── pubspec.yaml
```

```dart
// lib/src/models/todo.dart
class Todo {
  final int id;
  final String content;
  bool completed;
  final DateTime createdAt;

  Todo({
    required this.id,
    required this.content,
    this.completed = false,
    DateTime? createdAt,
  }) : createdAt = createdAt ?? DateTime.now();

  factory Todo.fromJson(Map<String, dynamic> json) => Todo(
    id: json['id'] as int,
    content: json['content'] as String,
    completed: json['completed'] as bool? ?? false,
    createdAt: json['createdAt'] != null
      ? DateTime.parse(json['createdAt'] as String) : null,
  );

  Map<String, dynamic> toJson() => {
    'id': id, 'content': content,
    'completed': completed,
    'createdAt': createdAt.toIso8601String(),
  };
}
```

```dart
// lib/todo_app.dart — 统一导出
export 'src/models/todo.dart';
export 'src/repository.dart';
export 'src/exceptions.dart';
export 'src/commands.dart';
```

```dart
// bin/todo_app.dart — 入口
import 'package:todo_app/todo_app.dart';

void main() async {
  var repo = TodoRepository('todos.json');
  await repo.load();
  runCommands(repo);
}
```
