## 一、Flutter 导航模型

Flutter 的导航基于**栈**结构——页面压栈（push）和出栈（pop），类似浏览器的"前进/后退"。

```mermaid
flowchart LR
    A[首页] -->|push| B[列表页]
    B -->|push| C[详情页]
    C -->|pop| B
    B -->|pop| A
```

Flutter 提供两套路由 API：

| | Navigator 1.0 | GoRouter（推荐） |
|---|---------------|-----------------|
| 风格 | 命令式（手动 push/pop） | 声明式（定义路由表） |
| 复杂度 | 简单场景够用 | 支持嵌套、重定向、深链接 |
| Web 支持 | URL 不同步 | URL 自动同步 |
| 学习成本 | 低 | 中 |
| 官方推荐 | 简单 App | 生产级 App |

## 二、Navigator 1.0：基础导航

### 2.1 push 和 pop

```dart
// 跳转到新页面
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const DetailPage()),
);

// 返回上一页
Navigator.pop(context);

// 带返回值 pop
Navigator.pop(context, 'result_value');
```

### 2.2 完整的页面跳转示例

```dart
// 首页 — 跳转到详情页并接收返回值
class HomePage extends StatelessWidget {
  void _openDetail(BuildContext context) async {
    // push 返回 Future，可以 await 返回值
    final result = await Navigator.push<String>(
      context,
      MaterialPageRoute(
        builder: (context) => const DetailPage(title: '我的日记'),
      ),
    );
    if (result != null) {
      print('返回结果: $result');
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('首页')),
      body: Center(
        child: ElevatedButton(
          onPressed: () => _openDetail(context),
          child: const Text('打开详情'),
        ),
      ),
    );
  }
}

// 详情页 — 返回结果
class DetailPage extends StatelessWidget {
  final String title;

  const DetailPage({super.key, required this.title});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(title)),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.pop(context, '已保存');
          },
          child: const Text('保存并返回'),
        ),
      ),
    );
  }
}
```

### 2.3 命名路由

对于页面较多的小型应用，可以用命名路由统一管理：

```dart
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (context) => const HomePage(),
    '/detail': (context) => const DetailPage(),
    '/settings': (context) => const SettingsPage(),
  },
);

// 跳转
Navigator.pushNamed(context, '/detail');

// 带参数跳转
Navigator.pushNamed(
  context,
  '/detail',
  arguments: {'id': '123', 'title': '我的日记'},
);

// 接收参数（在目标页面）
@override
Widget build(BuildContext context) {
  final args = ModalRoute.of(context)!.settings.arguments as Map<String, String>;
  final id = args['id'];
  // ...
}
```

**命名路由的局限：**
- 参数没有类型安全（arguments 是 Object?）
- 不支持深链接
- Web 端 URL 不会自动更新
- 复杂的嵌套路由难以管理

> **结论**：简单 App 用命名路由，生产级 App 用 GoRouter。

### 2.4 常用导航方法

```dart
// 替换当前页面（不保留当前页在栈中）
Navigator.pushReplacement(
  context,
  MaterialPageRoute(builder: (context) => const LoginPage()),
);

// 清空栈并跳转（适合登录后跳首页）
Navigator.pushAndRemoveUntil(
  context,
  MaterialPageRoute(builder: (context) => const HomePage()),
  (route) => false,  // 移除所有之前的路由
);

// 返回到栈中某个特定页面
Navigator.popUntil(context, (route) => route.isFirst);

// 检查是否可以 pop（避免在根页面 pop 导致应用退出）
if (Navigator.canPop(context)) {
  Navigator.pop(context);
}
```

## 三、GoRouter：声明式路由（推荐）

GoRouter 是 Flutter 官方推荐的路由库，解决了 Navigator 1.0 的所有痛点。

### 3.1 基本配置

```yaml
# pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  go_router: ^14.0.0
```

```dart
import 'package:go_router/go_router.dart';

// 定义路由表
final router = GoRouter(
  initialLocation: '/',
  routes: [
    GoRoute(
      path: '/',
      builder: (context, state) => const HomePage(),
    ),
    GoRoute(
      path: '/detail/:id',  // 路径参数
      builder: (context, state) {
        final id = state.pathParameters['id']!;
        return DetailPage(id: id);
      },
    ),
    GoRoute(
      path: '/settings',
      builder: (context, state) => const SettingsPage(),
    ),
  ],
);

// 在 MaterialApp 中使用
MaterialApp.router(
  routerConfig: router,
)
```

### 3.2 导航方法

```dart
// 跳转
context.go('/detail/123');       // 替换当前路由（无动画）
context.push('/detail/123');     // 压栈（有动画，推荐）

// 带查询参数
context.push(Uri(
  path: '/detail/123',
  queryParameters: {'from': 'home'},
).toString());

// 或使用命名路由
context.goNamed('detail', pathParameters: {'id': '123'});

// 返回
context.pop();

// 带返回值返回
context.pop('result');

// 替换当前路由
context.replace('/detail/123');

// 接收返回值
final result = await context.push<String>('/detail/123');
```

### 3.3 路径参数与查询参数

```dart
GoRoute(
  path: '/journal/:id',  // :id 是路径参数
  builder: (context, state) {
    // 路径参数
    final id = state.pathParameters['id']!;

    // 查询参数
    final from = state.uri.queryParameters['from'];

    // 额外参数（类型安全）
    final extra = state.extra as Journal?;

    return DetailPage(id: id, from: from, journal: extra);
  },
),

// 跳转方式
context.push('/journal/123?from=home');
context.push('/journal/123', extra: journal);  // 传递对象
```

### 3.4 嵌套路由（Shell Route）

嵌套路由是 GoRouter 最强大的功能——在多个页面间共享同一个布局（如底部导航栏）：

```dart
final router = GoRouter(
  initialLocation: '/',
  routes: [
    ShellRoute(
      builder: (context, state, child) {
        // 共享的底部导航布局
        return ScaffoldWithNavBar(child: child);
      },
      routes: [
        GoRoute(
          path: '/',
          builder: (context, state) => const HomePage(),
        ),
        GoRoute(
          path: '/explore',
          builder: (context, state) => const ExplorePage(),
        ),
        GoRoute(
          path: '/profile',
          builder: (context, state) => const ProfilePage(),
        ),
      ],
    ),
    // 不在底部导航内的页面（如详情页）
    GoRoute(
      path: '/journal/:id',
      builder: (context, state) {
        final id = state.pathParameters['id']!;
        return DetailPage(id: id);
      },
    ),
  ],
);

// 带底部导航的 Scaffold
class ScaffoldWithNavBar extends StatelessWidget {
  final Widget child;

  const ScaffoldWithNavBar({super.key, required this.child});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: child,
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: _currentIndex(context),
        onTap: (index) => _onItemTapped(context, index),
        items: const [
          BottomNavigationBarItem(icon: Icon(Icons.home), label: '首页'),
          BottomNavigationBarItem(icon: Icon(Icons.explore), label: '发现'),
          BottomNavigationBarItem(icon: Icon(Icons.person), label: '我的'),
        ],
      ),
    );
  }

  int _currentIndex(BuildContext context) {
    final location = GoRouterState.of(context).uri.path;
    if (location.startsWith('/explore')) return 1;
    if (location.startsWith('/profile')) return 2;
    return 0;
  }

  void _onItemTapped(BuildContext context, int index) {
    switch (index) {
      case 0: context.go('/');
      case 1: context.go('/explore');
      case 2: context.go('/profile');
    }
  }
}
```

### 3.5 重定向（Redirect）

重定向用于权限控制，如未登录时跳转到登录页：

```dart
final router = GoRouter(
  redirect: (context, state) {
    final isLoggedIn = AuthService.isLoggedIn;
    final isLoginRoute = state.matchedLocation == '/login';

    // 未登录且不在登录页 → 跳转登录页
    if (!isLoggedIn && !isLoginRoute) return '/login';

    // 已登录且在登录页 → 跳转首页
    if (isLoggedIn && isLoginRoute) return '/';

    // 不需要重定向
    return null;
  },
  routes: [
    GoRoute(path: '/', builder: (context, state) => const HomePage()),
    GoRoute(path: '/login', builder: (context, state) => const LoginPage()),
    // ...
  ],
);
```

## 四、Flutter Journal 的路由配置

把前面学到的知识综合起来，为 Flutter Journal 配置完整的路由系统：

```dart
import 'package:go_router/go_router.dart';

// 路由路径常量
class AppRoutes {
  static const home = '/';
  static const detail = '/journal/:id';
  static const editor = '/editor';
  static const settings = '/settings';
  static const login = '/login';
}

// 路由配置
final router = GoRouter(
  initialLocation: AppRoutes.home,
  redirect: (context, state) {
    final isLoggedIn = AuthService.instance.isLoggedIn;
    final isLoginRoute = state.matchedLocation == AppRoutes.login;

    if (!isLoggedIn && !isLoginRoute) return AppRoutes.login;
    if (isLoggedIn && isLoginRoute) return AppRoutes.home;
    return null;
  },
  routes: [
    ShellRoute(
      builder: (context, state, child) => MainScaffold(child: child),
      routes: [
        GoRoute(
          path: AppRoutes.home,
          name: 'home',
          builder: (context, state) => const HomePage(),
        ),
        GoRoute(
          path: '/explore',
          name: 'explore',
          builder: (context, state) => const ExplorePage(),
        ),
        GoRoute(
          path: AppRoutes.settings,
          name: 'settings',
          builder: (context, state) => const SettingsPage(),
        ),
      ],
    ),
    GoRoute(
      path: AppRoutes.detail,
      name: 'detail',
      builder: (context, state) {
        final id = state.pathParameters['id']!;
        return DetailPage(journalId: id);
      },
    ),
    GoRoute(
      path: AppRoutes.editor,
      name: 'editor',
      builder: (context, state) {
        final journal = state.extra as Journal?;
        return EditorPage(journal: journal);
      },
    ),
    GoRoute(
      path: AppRoutes.login,
      name: 'login',
      builder: (context, state) => const LoginPage(),
    ),
  ],
);
```

## 五、深链接

深链接（Deep Link）让用户通过 URL 直接打开 App 内的特定页面，支持 Web URL 同步和移动端 App Link/Universal Link。

### 5.1 Web 端 URL 策略

```dart
// 默认使用 # 号路由（如 example.com/#/detail/123）
// 去掉 # 号，使用真实路径（如 example.com/detail/123）

// 在 main() 中添加
import 'package:flutter_web_plugins/url_strategy.dart';

void main() {
  usePathUrlStrategy();  // 使用路径策略，去掉 # 号
  runApp(const JournalApp());
}
```

### 5.2 Android 深链接

在 `android/app/src/main/AndroidManifest.xml` 中添加 intent filter：

```xml
<activity android:name=".MainActivity">
  <intent-filter>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="https" android:host="yourdomain.com"/>
  </intent-filter>

  <!-- 自定义 scheme -->
  <intent-filter>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="flutterjournal"/>
  </intent-filter>
</activity>
```

### 5.3 iOS 深链接

在 `ios/Runner/Info.plist` 中添加：

```xml
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleTypeRole</key>
    <string>Viewer</string>
    <key>CFBundleURLName</key>
    <string>com.example.flutterjournal</string>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>flutterjournal</string>
    </array>
  </dict>
</array>
```

### 5.4 测试深链接

```bash
# Android
adb shell am start -a android.intent.action.VIEW -d "flutterjournal://journal/123"

# iOS
xcrun simctl openurl booted "flutterjournal://journal/123"

# Web — 直接在浏览器地址栏输入
# https://yourdomain.com/journal/123
```

## 六、导航动画自定义

```dart
// GoRouter 自定义页面切换动画
GoRoute(
  path: '/detail/:id',
  pageBuilder: (context, state) {
    final id = state.pathParameters['id']!;
    return CustomTransitionPage(
      key: state.pageKey,
      child: DetailPage(id: id),
      transitionsBuilder: (context, animation, secondaryAnimation, child) {
        // 从右滑入
        return SlideTransition(
          position: Tween<Offset>(
            begin: const Offset(1, 0),
            end: Offset.zero,
          ).animate(CurvedAnimation(
            parent: animation,
            curve: Curves.easeInOut,
          )),
          child: child,
        );
      },
    );
  },
),

// 常用动画效果
// 淡入淡出
FadeTransition(opacity: animation, child: child)

// 缩放
ScaleTransition(scale: animation, child: child)

// 从底部滑入
SlideTransition(
  position: Tween<Offset>(
    begin: const Offset(0, 1),
    end: Offset.zero,
  ).animate(animation),
  child: child,
)
```

## 七、小结

| 概念 | 核心要点 |
|------|---------|
| Navigator 1.0 | push/pop 栈式导航，简单场景够用 |
| 命名路由 | 统一管理路由表，但参数不类型安全 |
| GoRouter | 声明式路由，支持嵌套、重定向、深链接 |
| ShellRoute | 共享布局（底部导航），嵌套路由 |
| 深链接 | Web URL 同步 + 移动端 App Link |
| 自定义动画 | pageBuilder + CustomTransitionPage |
