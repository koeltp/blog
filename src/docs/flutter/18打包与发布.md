## 一、发布前检查清单

- [ ] 移除所有 `print()` 和 `debugPrint()`
- [ ] 关闭 `debugShowCheckedModeBanner: false`
- [ ] 确认 `flutter analyze` 无错误
- [ ] 确认 `flutter test` 全部通过
- [ ] 更新 `pubspec.yaml` 中的版本号
- [ ] 准备应用图标和启动屏
- [ ] 准备商店截图和描述

## 二、Android 打包

### 2.1 签名配置

```bash
# 生成签名密钥
keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload
```

```properties
# android/key.properties
storePassword=你的密码
keyPassword=你的密码
keyAlias=upload
storeFile=../../upload-keystore.jks
```

```groovy
// android/app/build.gradle
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 2.2 构建 AAB

```bash
# 构建 App Bundle（推荐，Google Play 要求）
flutter build appbundle --release

# 构建 APK
flutter build apk --release

# 按架构拆分 APK（更小体积）
flutter build apk --split-per-abi
```

### 2.3 上架 Google Play

1. 创建 [Google Play Console](https://play.google.com/console) 账号
2. 创建应用，填写商店信息
3. 上传 AAB 到"内部测试"轨道
4. 填写内容分级、隐私政策等
5. 提交审核

## 三、iOS 打包

### 3.1 证书与配置

1. Apple Developer 账号（$99/年）
2. 在 [Apple Developer](https://developer.apple.com) 创建：
   - App ID
   - Distribution Certificate
   - Provisioning Profile

### 3.2 Xcode 配置

```bash
# 打开 Xcode 工作区
open ios/Runner.xcworkspace
```

在 Xcode 中：
- Signing & Capabilities → 选择 Team 和 Bundle Identifier
- General → 设置 Version 和 Build 号
- App Icons → 设置应用图标

### 3.3 构建与上传

```bash
# 构建 IPA
flutter build ipa --release

# 上传到 App Store Connect
xcrun altool --upload-app --type ios --file build/ios/ipa/journal.ipa --apiKey YOUR_KEY --apiIssuer YOUR_ISSUER
```

或通过 Xcode → Product → Archive → Distribute App 上传。

## 五、CI/CD 自动化

### 5.1 GitHub Actions

```yaml
# .github/workflows/build.yml
name: Build and Deploy

on:
  push:
    tags: ['v*']

jobs:
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.x'
          channel: 'stable'
      - run: flutter pub get
      - run: flutter test
      - run: flutter build appbundle --release
      - uses: actions/upload-artifact@v4
        with:
          name: app-release.aab
          path: build/app/outputs/bundle/release/app-release.aab

  build-ios:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.x'
      - run: flutter pub get
      - run: flutter test
      - run: flutter build ipa --release --no-codesign
      - uses: actions/upload-artifact@v4
        with:
          name: runner.ipa
          path: build/ios/ipa/*.ipa
```

### 5.2 Fastlane（可选）

```bash
# 安装
gem install fastlane

# 初始化
cd android && fastlane init
cd ios && fastlane init
```

```ruby
# android/fastlane/Fastfile
lane :release do
  sh("flutter build appbundle --release")
  upload_to_play_store(
    track: 'internal',
    aab: '../build/app/outputs/bundle/release/app-release.aab',
  )
end
```

## 六、版本号管理

```yaml
# pubspec.yaml
version: 1.2.0+3  # 1.2.0 是版本名，+3 是构建号
```

- 版本名（1.2.0）：用户可见的版本号
- 构建号（3）：每次上传递增，商店用来区分不同构建

## 七、小结

| 平台 | 产物 | 上架方式 |
|------|------|---------|
