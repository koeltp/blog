# 类与面向对象

> 上一章我们实现了命令解析器。本章学习 TypeScript 的面向对象编程——类、访问修饰符、抽象类、接口实现，然后用这些知识封装 TaskManager 类。

TypeScript 的类在 ES6 类的基础上增加了访问修饰符、参数属性、抽象类等特性，让 OOP 更加严谨。

## 类基础

### 声明与实例化

```typescript
class Point {
  x: number
  y: number

  constructor(x: number, y: number) {
    this.x = x
    this.y = y
  }

  distanceTo(other: Point): number {
    return Math.sqrt((this.x - other.x) ** 2 + (this.y - other.y) ** 2)
  }
}

const p1 = new Point(1, 2)
const p2 = new Point(4, 6)
console.log(p1.distanceTo(p2))  // 5
```

### 参数属性 — 简化构造函数

在构造函数参数前加访问修饰符，TypeScript 自动创建并赋值同名属性：

```typescript
class Point {
  constructor(
    public x: number,
    public y: number,
  ) {}
}

const p = new Point(1, 2)
console.log(p.x, p.y)  // 1 2
```

等价于手动声明 + 赋值，但代码更简洁。

## 访问修饰符

### public（默认）

任何地方都能访问：

```typescript
class User {
  public name: string  // public 可省略，默认就是 public
  constructor(name: string) {
    this.name = name
  }
}
```

### private — 仅类内部访问

```typescript
class BankAccount {
  private balance: number = 0

  deposit(amount: number): void {
    if (amount > 0) this.balance += amount
  }

  getBalance(): number {
    return this.balance
  }
}

const account = new BankAccount()
account.deposit(100)
// account.balance  // ❌ 私有属性，外部不可访问
console.log(account.getBalance())  // 100
```

### protected — 类内部 + 子类可访问

```typescript
class Animal {
  constructor(protected name: string) {}

  protected log(msg: string): void {
    console.log(`[${this.name}] ${msg}`)
  }
}

class Dog extends Animal {
  constructor(name: string) {
    super(name)
  }

  bark(): void {
    this.log('汪汪！')  // ✅ 子类可以访问 protected 成员
  }
}

const dog = new Dog('旺财')
dog.bark()  // [旺财] 汪汪！
// dog.name  // ❌ 外部不可访问
```

### # — ES 原生私有字段

```typescript
class Secret {
  #value: string  // ES2022 原生私有字段

  constructor(value: string) {
    this.#value = value
  }
}

const s = new Secret('hello')
// s.#value  // ❌ 语法错误，无法访问
```

`#` 和 `private` 的区别：

| 特性 | `private` | `#` |
|------|----------|-----|
| 编译时检查 | ✅ | ✅ |
| 运行时隔离 | ❌（编译后可访问） | ✅（运行时也不可访问） |
| 类型系统可见 | ✅（可被 typeof 提取） | ❌ |

> 库开发建议用 `#`，确保运行时也安全；应用开发用 `private` 就够了。

## readonly 修饰符

```typescript
class Task {
  constructor(
    public readonly id: number,
    public title: string,
  ) {}
}

const task = new Task(1, '学习')
task.title = '复习'  // ✅
// task.id = 2        // ❌ 只读
```

`readonly` 只在类型检查阶段生效，运行时仍可修改（除非用 `Object.freeze`）。

## 存取器 — getter / setter

```typescript
class Temperature {
  private _celsius: number = 0

  get celsius(): number {
    return this._celsius
  }

  set celsius(value: number) {
    if (value < -273.15) {
      throw new Error('温度不能低于绝对零度')
    }
    this._celsius = value
  }

  get fahrenheit(): number {
    return this._celsius * 1.8 + 32
  }

  set fahrenheit(value: number) {
    this.celsius = (value - 32) / 1.8
  }
}

const temp = new Temperature()
temp.celsius = 25
console.log(temp.fahrenheit)  // 77
```

> getter 只有 get 没有 set 时，属性自动变为 readonly。

## 继承

### extends — 类继承

```typescript
class BaseEntity {
  constructor(
    public readonly id: number,
    public createdAt: Date = new Date(),
  ) {}

  toString(): string {
    return `#${this.id}`
  }
}

class Task extends BaseEntity {
  constructor(
    id: number,
    public title: string,
    public status: string = 'todo',
  ) {
    super(id)  // 必须调用 super()
  }

  // 覆盖父类方法
  toString(): string {
    return `${super.toString()} ${this.title} [${this.status}]`
  }
}
```

### implements — 接口实现

```typescript
interface Serializable {
  serialize(): string
}

interface Comparable<T> {
  compareTo(other: T): number
}

class Task implements Serializable, Comparable<Task> {
  constructor(public id: number, public title: string) {}

  serialize(): string {
    return JSON.stringify({ id: this.id, title: this.title })
  }

  compareTo(other: Task): number {
    return this.id - other.id
  }
}
```

> `implements` 只检查类的实例端是否满足接口，不改变类的实现。

### extends vs implements

```typescript
// extends：继承实现（代码复用）
class Dog extends Animal { ... }

// implements：实现契约（类型检查）
class Task implements Serializable { ... }

// 可以同时使用
class AdvancedTask extends Task implements Serializable { ... }
```

## 抽象类

抽象类不能被实例化，只能被继承，用于定义公共行为：

```typescript
abstract class Repository<T> {
  protected items: T[] = []

  abstract findById(id: number): T | undefined

  findAll(): T[] {
    return [...this.items]
  }

  add(item: T): void {
    this.items.push(item)
  }

  remove(id: number): boolean {
    const index = this.items.findIndex(item => this.getItemId(item) === id)
    if (index === -1) return false
    this.items.splice(index, 1)
    return true
  }

  protected abstract getItemId(item: T): number
}
```

## static — 静态成员

```typescript
class Task {
  private static nextId = 1

  static create(title: string): Task {
    return new Task(Task.nextId++, title)
  }

  private constructor(
    public readonly id: number,
    public title: string,
  ) {}
}

// const task = new Task(1, 'test')  // ❌ 构造函数是 private
const task = Task.create('学习 TS')  // ✅ 只能通过工厂方法创建
```

> 私有构造函数 + 静态工厂方法是一种常见的创建模式，可以控制实例化过程。

## 实战：封装 TaskManager 类

```typescript
// ===== 类型定义 =====

type TaskStatus = 'todo' | 'in-progress' | 'done'
type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

interface ITask {
  readonly id: number
  title: string
  description: string
  status: TaskStatus
  priority: TaskPriority
  tags: readonly string[]
  readonly createdAt: Date
  updatedAt: Date
}

interface ITaskManager {
  add(title: string, priority?: TaskPriority): ITask
  remove(id: number): boolean
  update(id: number, changes: Partial<Pick<ITask, 'title' | 'description' | 'status' | 'priority'>>): ITask | undefined
  find(id: number): ITask | undefined
  list(filter?: { status?: TaskStatus; priority?: TaskPriority }): readonly ITask[]
  count(): number
}

// ===== 实现 =====

class Task implements ITask {
  static #nextId = 1

  readonly id: number
  title: string
  description: string = ''
  status: TaskStatus = 'todo'
  priority: TaskPriority
  #tags: string[] = []
  readonly createdAt: Date
  updatedAt: Date

  constructor(title: string, priority: TaskPriority = 'medium') {
    this.id = Task.#nextId++
    this.title = title
    this.priority = priority
    this.createdAt = new Date()
    this.updatedAt = this.createdAt
  }

  get tags(): readonly string[] {
    return Object.freeze([...this.#tags])
  }

  addTag(tag: string): void {
    if (!this.#tags.includes(tag)) {
      this.#tags.push(tag)
      this.updatedAt = new Date()
    }
  }

  removeTag(tag: string): void {
    const index = this.#tags.indexOf(tag)
    if (index !== -1) {
      this.#tags.splice(index, 1)
      this.updatedAt = new Date()
    }
  }
}

class TaskManager implements ITaskManager {
  #tasks: Map<number, ITask> = new Map()

  add(title: string, priority: TaskPriority = 'medium'): ITask {
    const task = new Task(title, priority)
    this.#tasks.set(task.id, task)
    return task
  }

  remove(id: number): boolean {
    return this.#tasks.delete(id)
  }

  update(id: number, changes: Partial<Pick<ITask, 'title' | 'description' | 'status' | 'priority'>>): ITask | undefined {
    const task = this.#tasks.get(id)
    if (!task) return undefined

    Object.assign(task, changes)
    task.updatedAt = new Date()
    return task
  }

  find(id: number): ITask | undefined {
    return this.#tasks.get(id)
  }

  list(filter?: { status?: TaskStatus; priority?: TaskPriority }): readonly ITask[] {
    let tasks = [...this.#tasks.values()]
    if (filter?.status) {
      tasks = tasks.filter(t => t.status === filter.status)
    }
    if (filter?.priority) {
      tasks = tasks.filter(t => t.priority === filter.priority)
    }
    return Object.freeze(tasks)
  }

  count(): number {
    return this.#tasks.size
  }
}

// ===== 测试 =====

const manager = new TaskManager()
const t1 = manager.add('学习 TypeScript', 'high')
const t2 = manager.add('写代码', 'medium')
t1.addTag('学习')

manager.update(t1.id, { status: 'in-progress' })

console.log(manager.list())
console.log(manager.list({ status: 'todo' }))
console.log(manager.count())
```
