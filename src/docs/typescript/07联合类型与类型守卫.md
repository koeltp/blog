# 联合类型与类型守卫

> 上一章我们用泛型构建了通用 Repository。本章学习 TypeScript 的类型组合与窄化——联合/交叉类型、类型守卫、可辨识联合，实现灵活的多类型任务调度。

联合类型让一个值可以是多种类型之一；类型守卫让 TypeScript 在特定代码路径中"收窄"类型。这是 TS 类型系统最实用的部分。

## 联合类型

### 基本用法

```typescript
let id: number | string = 123
id = 'ABC-001'  // ✅

// 函数参数
function formatId(id: number | string): string {
  // 这里只能调用 number 和 string 的公共方法
  return id.toString()
}
```

### 对象联合类型

```typescript
interface Success {
  success: true
  data: string
}

interface Failure {
  success: false
  error: string
}

type Result = Success | Failure

function handleResult(result: Result) {
  if (result.success) {
    console.log(result.data)   // ✅ 类型为 Success
  } else {
    console.log(result.error)  // ✅ 类型为 Failure
  }
}
```

## 交叉类型

交叉类型把多个类型合并为一个：

```typescript
interface Timestamped {
  createdAt: Date
  updatedAt: Date
}

interface Versioned {
  version: number
}

type Entity = Timestamped & Versioned & {
  id: number
}

const entity: Entity = {
  id: 1,
  createdAt: new Date(),
  updatedAt: new Date(),
  version: 1,
}
```

### 交叉的陷阱

同名属性类型不同时，交叉结果为 `never`：

```typescript
type A = { name: string }
type B = { name: number }
type C = A & B
// C 的 name 属性类型为 string & number = never
```

## 类型窄化（Narrowing）

TypeScript 根据代码逻辑自动收窄类型：

### typeof

```typescript
function pad(value: string | number, padding: string | number): string {
  if (typeof padding === 'number') {
    return ' '.repeat(padding) + value  // padding: number
  }
  return padding + value  // padding: string
}
```

`typeof` 能识别的类型：`'string'`、`'number'`、`'boolean'`、`'symbol'`、`'bigint'`、`'undefined'`、`'object'`、`'function'`。

> `typeof null === 'object'`，不能用来区分 null。

### 真值窄化

```typescript
function printLength(value: string | null) {
  if (value) {
    console.log(value.length)  // value: string（排除了 null 和 ''）
  }
}
```

### 相等性窄化

```typescript
function compare(a: string | number, b: string | boolean) {
  if (a === b) {
    // a 和 b 都是 string（唯一交集）
    a.toUpperCase()  // ✅
    b.toUpperCase()  // ✅
  }
}
```

### in 操作符

```typescript
interface Fish { swim(): void }
interface Bird { fly(): void }

function move(animal: Fish | Bird) {
  if ('swim' in animal) {
    animal.swim()  // animal: Fish
  } else {
    animal.fly()   // animal: Bird
  }
}
```

### instanceof

```typescript
function formatValue(value: Date | string) {
  if (value instanceof Date) {
    return value.toISOString()  // value: Date
  }
  return value.toUpperCase()    // value: string
}
```

### 赋值窄化

```typescript
let value: string | number
value = 'hello'
value.toUpperCase()  // ✅ 赋值后 TS 知道是 string
value = 42
value.toFixed(2)     // ✅ 赋值后 TS 知道是 number
```

## 类型谓词 — 自定义类型守卫

当内置窄化不够用时，用 `is` 关键字定义自己的类型守卫：

```typescript
interface Dog { bark(): void }
interface Cat { meow(): void }

function isDog(animal: Dog | Cat): animal is Dog {
  return 'bark' in animal
}

function interact(animal: Dog | Cat) {
  if (isDog(animal)) {
    animal.bark()   // ✅ animal: Dog
  } else {
    animal.meow()   // ✅ animal: Cat
  }
}
```

### 复杂类型谓词

```typescript
type ApiResponse<T> =
  | { success: true; data: T }
  | { success: false; error: string }

function isSuccess<T>(response: ApiResponse<T>): response is { success: true; data: T } {
  return response.success === true
}

function handleResponse(response: ApiResponse<string>) {
  if (isSuccess(response)) {
    console.log(response.data.toUpperCase())  // ✅
  } else {
    console.log(response.error)               // ✅
  }
}
```

### 断言函数

`asserts` 关键字表示"如果函数正常返回，则条件一定成立"：

```typescript
function assertString(value: unknown): asserts value is string {
  if (typeof value !== 'string') {
    throw new TypeError(`Expected string, got ${typeof value}`)
  }
}

const input: unknown = 'hello'
assertString(input)
input.toUpperCase()  // ✅ 此后 input 类型为 string
```

## 可辨识联合（Discriminated Union）

这是 TypeScript 中最重要的模式之一——用公共属性区分子类型：

```typescript
// 可辨识标签：type 字段
interface Rectangle {
  type: 'rectangle'
  width: number
  height: number
}

interface Circle {
  type: 'circle'
  radius: number
}

interface Triangle {
  type: 'triangle'
  base: number
  height: number
}

type Shape = Rectangle | Circle | Triangle

function area(shape: Shape): number {
  switch (shape.type) {
    case 'rectangle':
      return shape.width * shape.height      // ✅ shape: Rectangle
    case 'circle':
      return Math.PI * shape.radius ** 2     // ✅ shape: Circle
    case 'triangle':
      return 0.5 * shape.base * shape.height  // ✅ shape: Triangle
    default:
      // 穷举检查：如果 Shape 增加新类型，这里会报错
      const _exhaustive: never = shape
      throw new Error(`Unknown shape: ${_exhaustive}`)
  }
}
```

可辨识联合的优势：

- **完备性检查** — `never` + `default` 确保处理所有情况
- **类型安全** — 每个分支自动窄化到具体类型
- **扩展安全** — 新增子类型后，编译器会提醒你补充分支

## 实战：多类型任务调度

用可辨识联合实现不同类型任务的调度系统：

```typescript
// ===== 任务类型定义 =====

type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

// 可辨识标签：kind 字段
interface OnceTask {
  kind: 'once'
  id: number
  title: string
  priority: TaskPriority
  dueDate?: Date
  done: boolean
}

interface RecurringTask {
  kind: 'recurring'
  id: number
  title: string
  priority: TaskPriority
  interval: 'daily' | 'weekly' | 'monthly'
  nextRun: Date
}

interface ScheduledTask {
  kind: 'scheduled'
  id: number
  title: string
  priority: TaskPriority
  cron: string
  enabled: boolean
}

type Task = OnceTask | RecurringTask | ScheduledTask

// ===== 调度逻辑 =====

function execute(task: Task): string {
  switch (task.kind) {
    case 'once':
      task.done = true
      return `执行一次性任务: ${task.title}`
    case 'recurring':
      task.nextRun = calcNextRun(task.nextRun, task.interval)
      return `执行周期任务: ${task.title}（下次: ${task.nextRun.toDateString()}）`
    case 'scheduled':
      if (!task.enabled) return `跳过已禁用的计划任务: ${task.title}`
      return `执行计划任务: ${task.title}（cron: ${task.cron}）`
    default:
      const _exhaustive: never = task
      throw new Error(`未知任务类型: ${_exhaustive}`)
  }
}

function calcNextRun(current: Date, interval: RecurringTask['interval']): Date {
  const next = new Date(current)
  switch (interval) {
    case 'daily': next.setDate(next.getDate() + 1); break
    case 'weekly': next.setDate(next.getDate() + 7); break
    case 'monthly': next.setMonth(next.getMonth() + 1); break
  }
  return next
}

// ===== 任务管理 =====

class TaskScheduler {
  #tasks: Task[] = []

  add(task: Task): void {
    this.#tasks.push(task)
  }

  // 按优先级过滤
  getUrgent(): Task[] {
    return this.#tasks.filter(t => t.priority === 'urgent')
  }

  // 执行所有可执行任务
  executeAll(): string[] {
    return this.#tasks.map(execute)
  }

  // 类型守卫：获取特定类型的任务
  getOnceTasks(): OnceTask[] {
    return this.#tasks.filter((t): t is OnceTask => t.kind === 'once')
  }

  getRecurringTasks(): RecurringTask[] {
    return this.#tasks.filter((t): t is RecurringTask => t.kind === 'recurring')
  }
}

// ===== 测试 =====

const scheduler = new TaskScheduler()
scheduler.add({ kind: 'once', id: 1, title: '写周报', priority: 'high', done: false })
scheduler.add({ kind: 'recurring', id: 2, title: '站会', priority: 'medium', interval: 'daily', nextRun: new Date() })
scheduler.add({ kind: 'scheduled', id: 3, title: '部署', priority: 'high', cron: '0 2 * * *', enabled: true })

const results = scheduler.executeAll()
results.forEach(console.log)
```
