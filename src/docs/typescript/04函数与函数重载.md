# 函数与函数重载

> 上一章我们学会了用接口和类型别名描述数据形状。本章深入 TypeScript 函数——参数模式、返回值推断、重载签名，并用这些知识实现 Task Manager 的命令解析器。

函数是 JavaScript 的一等公民，TypeScript 在此基础上增加了完整的类型签名能力。

## 函数声明

### 两种写法

```typescript
// 函数声明
function add(a: number, b: number): number {
  return a + b
}

// 函数表达式
const add = function (a: number, b: number): number {
  return a + b
}

// 箭头函数
const add = (a: number, b: number): number => a + b
```

### 返回值类型

简单函数可以省略返回值类型，让 TS 推断：

```typescript
// 推断返回 number
function add(a: number, b: number) {
  return a + b
}

// void 表示不返回有意义的值
function log(msg: string): void {
  console.log(msg)
}

// never 表示永远不会返回
function fail(msg: string): never {
  throw new Error(msg)
}
```

## 参数模式

### 可选参数

```typescript
function greet(name: string, greeting?: string): string {
  return `${greeting || 'Hello'}, ${name}!`
}

greet('Alice')             // "Hello, Alice!"
greet('Alice', 'Hi')      // "Hi, Alice!"
```

> 可选参数必须在必选参数后面。

### 默认参数

```typescript
function greet(name: string, greeting: string = 'Hello'): string {
  return `${greeting}, ${name}!`
}

greet('Alice')             // "Hello, Alice!"
greet('Alice', 'Hi')      // "Hi, Alice!"
```

默认参数不需要 `?`，因为总有值。默认参数不影响参数类型：

```typescript
// greeting 的类型是 string，不是 string | undefined
```

### 剩余参数

```typescript
function sum(first: number, ...rest: number[]): number {
  return first + rest.reduce((a, b) => a + b, 0)
}

sum(1, 2, 3, 4)  // 10
```

### 解构参数

```typescript
interface Point {
  x: number
  y: number
}

// 对象解构 + 类型标注
function distance({ x, y }: Point): number {
  return Math.sqrt(x * x + y * y)
}

// 也可以内联类型
function move({ x, y }: { x: number; y: number }): Point {
  return { x: x + 1, y: y + 1 }
}
```

## 函数重载

当同一个函数需要支持不同的参数组合时，使用重载签名：

```typescript
// 重载签名
function parse(input: string): number
function parse(input: number): string
function parse(input: string | number): string | number {
  if (typeof input === 'string') {
    return Number(input)
  }
  return String(input)
}

const a = parse('42')   // 类型为 number
const b = parse(42)     // 类型为 string
```

### 重载顺序

TypeScript 从上到下匹配重载签名，**更具体的签名放在前面**：

```typescript
// ✅ 正确顺序
function makeDate(timestamp: number): Date
function makeDate(year: number, month: number, day: number): Date
function makeDate(yearOrTimestamp: number, month?: number, day?: number): Date {
  if (month !== undefined && day !== undefined) {
    return new Date(yearOrTimestamp, month - 1, day)
  }
  return new Date(yearOrTimestamp)
}

// ❌ 错误顺序——通用签名在前，具体签名永远不会匹配
function makeDate(year: number, month: number, day: number): Date
function makeDate(timestamp: number): Date
```

### 重载与回调

```typescript
// 事件监听：根据事件名决定回调参数类型
interface EventMap {
  click: { x: number; y: number }
  keydown: { key: string; code: number }
  resize: { width: number; height: number }
}

function on<K extends keyof EventMap>(
  event: K,
  callback: (payload: EventMap[K]) => void,
): void {
  // 实现...
}

on('click', ({ x, y }) => console.log(x, y))      // ✅ payload 类型为 { x: number; y: number }
on('keydown', ({ key }) => console.log(key))        // ✅ payload 类型为 { key: string; code: number }
// on('click', ({ key }) => ...)                     // ❌ click 事件没有 key
```

> 这种泛型 + 映射类型的模式比传统重载更灵活，是现代 TS 的推荐做法。

## this 类型

### 指定 this 类型

```typescript
interface DB {
  users: string[]
  log(this: DB, msg: string): void
}

const db: DB = {
  users: [],
  log(msg) {
    // this 类型为 DB
    console.log(`[${this.users.length} users] ${msg}`)
  },
}

db.log('init')  // ✅
const fn = db.log
// fn('init')   // ❌ 脱离上下文调用会报错
```

### 回调中的 this

```typescript
class Counter {
  count = 0

  // 箭头函数自动绑定 this
  increment = () => {
    this.count++
  }
}

const counter = new Counter()
const inc = counter.increment
inc()  // ✅ 箭头函数 this 绑定正确
```

## 函数类型体操

### 获取函数参数/返回值类型

```typescript
type Fn = (a: string, b: number) => boolean

type Params = Parameters<Fn>     // [a: string, b: number]
type Return = ReturnType<Fn>     // boolean
type FirstParam = Params[0]      // string
```

### 提取构造函数参数

```typescript
class Task {
  constructor(public title: string, public priority: string) {}
}

type TaskParams = ConstructorParameters<typeof Task>  // [title: string, priority: string]
```

## 实战：命令解析器

为 Task Manager 实现类型安全的命令解析器：

```typescript
// ===== 命令定义 =====

type Command =
  | { action: 'add'; title: string; priority?: string }
  | { action: 'done'; id: number }
  | { action: 'list'; filter?: string }
  | { action: 'delete'; id: number }
  | { action: 'help' }

// ===== 解析器 =====

function parseCommand(input: string): Command | null {
  const parts = input.trim().split(/\s+/)
  const action = parts[0]

  switch (action) {
    case 'add':
      return {
        action: 'add',
        title: parts.slice(1).join(' '),
        priority: 'medium',
      }
    case 'done':
      return { action: 'done', id: Number(parts[1]) }
    case 'list':
      return { action: 'list', filter: parts[1] }
    case 'delete':
      return { action: 'delete', id: Number(parts[1]) }
    case 'help':
      return { action: 'help' }
    default:
      console.log(`未知命令: ${action}，输入 help 查看帮助`)
      return null
  }
}

// ===== 命令处理器（重载） =====

function handleCommand(command: { action: 'add'; title: string; priority?: string }): string
function handleCommand(command: { action: 'done'; id: number }): string
function handleCommand(command: { action: 'list'; filter?: string }): string
function handleCommand(command: { action: 'delete'; id: number }): string
function handleCommand(command: { action: 'help' }): string
function handleCommand(command: Command): string {
  switch (command.action) {
    case 'add':
      return `添加任务: ${command.title}`
    case 'done':
      return `完成任务 #${command.id}`
    case 'list':
      return `列出任务${command.filter ? ` (筛选: ${command.filter})` : ''}`
    case 'delete':
      return `删除任务 #${command.id}`
    case 'help':
      return '可用命令: add, done, list, delete, help'
  }
}

// ===== 测试 =====

const input = 'add 学习 TypeScript'
const cmd = parseCommand(input)
if (cmd) {
  const result = handleCommand(cmd)  // 类型为 string
  console.log(result)                // 添加任务: 学习 TypeScript
}
```
