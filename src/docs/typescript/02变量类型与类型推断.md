# 变量、类型与类型推断

> 上一章我们搭建了环境并创建了 Task Manager 骨架。本章深入学习 TypeScript 的类型系统，然后用这些知识为项目定义更完善的数据模型。

TypeScript 的核心价值在于类型系统。理解类型如何工作，是写出可靠 TS 代码的基础。

## 基础类型

### JavaScript 原始类型

TypeScript 完整支持 JS 的原始类型：

```typescript
let isDone: boolean = false
let count: number = 42
let name: string = 'TypeScript'
let big: bigint = 100n

// null 和 undefined 是所有类型的子类型（strictNullChecks 关闭时）
let u: undefined = undefined
let n: null = null
```

### 数组

```typescript
let numbers: number[] = [1, 2, 3]
let strings: Array<string> = ['a', 'b', 'c']

// 只读数组，防止修改
const readonlyArr: ReadonlyArray<number> = [1, 2, 3]
// readonlyArr.push(4)  // ❌ 报错，不允许修改
```

### 元组

固定长度和类型的数组：

```typescript
// [id, title, done]
let task: [number, string, boolean] = [1, '学习 TS', false]

// 可选元素
let optional: [number, string?] = [1]
// 剩余元素
let rest: [number, ...string[]] = [1, 'a', 'b', 'c']
```

### 枚举

```typescript
enum Direction {
  Up,      // 0
  Down,    // 1
  Left,    // 2
  Right,   // 3
}

// 字符串枚举（推荐，可读性好）
enum Priority {
  Low = 'LOW',
  Medium = 'MEDIUM',
  High = 'HIGH',
}

// const 枚举（编译时内联，零运行时开销）
const enum Color {
  Red,
  Green,
  Blue,
}
let c = Color.Red  // 编译为 let c = 0
```

> 字符串枚举比数字枚举更安全——数字枚举可以接受任意数字，字符串枚举只能接受定义的值。

## 特殊类型

### any — 逃避类型检查

```typescript
let anything: any = 'hello'
anything = 42          // ✅ 任意赋值
anything.toUpperCase() // ✅ 任意调用（运行时可能报错）
```

`any` 完全关闭类型检查，等于回到 JavaScript。**能不用就不用**。

### unknown — 安全的 any

```typescript
let value: unknown = 'hello'

// value.toUpperCase()  // ❌ 不能直接使用，必须先窄化

if (typeof value === 'string') {
  console.log(value.toUpperCase())  // ✅ 窄化后可以
}
```

`unknown` 是类型安全的 `any`：可以接收任何值，但使用前必须检查类型。

### never — 永远不会有值

```typescript
// 函数抛出异常，永远不会正常返回
function throwError(msg: string): never {
  throw new Error(msg)
}

// 无限循环
function infiniteLoop(): never {
  while (true) {}
}

// 类型穷举检查
type Shape = 'circle' | 'square'

function area(shape: Shape): number {
  switch (shape) {
    case 'circle': return Math.PI
    case 'square': return 1
    default:
      // 如果 Shape 增加新值，这里会报类型错误
      const _exhaustive: never = shape
      return _exhaustive
  }
}
```

### void — 没有返回值

```typescript
function log(msg: string): void {
  console.log(msg)
  // 没有 return 语句，或 return undefined
}
```

`void` 和 `undefined` 的区别：`void` 表示"调用方不应该关心返回值"，`undefined` 表示"返回 undefined 这个值"。

## 类型推断

TypeScript 足够聪明，很多时候不需要你显式标注类型：

### 变量初始化推断

```typescript
let name = 'TypeScript'  // 推断为 string，不是 let name: string
let count = 42           // 推断为 number
let items = [1, 2, 3]    // 推断为 number[]

// ⚠️ 如果没有初始值，推断为 any（strict 模式下会报错）
let x  // 推断为 any
```

### 函数返回值推断

```typescript
// 推断返回值为 number
function add(a: number, b: number) {
  return a + b
}

// 推断返回值为 string[]
function getNames() {
  return ['Alice', 'Bob']
}
```

> 简单函数可以省略返回值类型，但公共 API 建议显式标注，方便阅读和避免推断错误。

### 最佳通用类型

```typescript
let arr = [1, 'hello', true]  // 推断为 (number | string | boolean)[]
```

## 类型断言

当你比 TypeScript 更了解类型时，可以用断言告诉编译器：

### as 语法（推荐）

```typescript
const input = document.getElementById('input') as HTMLInputElement
input.value  // ✅ 有类型提示
```

### 尖括号语法

```typescript
const input = <HTMLInputElement>document.getElementById('input')
```

> 在 JSX/TSX 中只能用 `as` 语法，尖括号会和标签语法冲突。

### 非空断言 !

```typescript
const value: string | null = getValue()
const length: number = value!.length  // 告诉 TS value 一定不为 null
```

> `!` 是危险的——如果运行时值为 `null`，就会报错。优先用类型守卫代替。

### 双重断言（极其危险）

```typescript
// ❌ 千万别这么写，绕过了所有类型检查
const s = 42 as unknown as string
```

## 字面量类型

比枚举更轻量的精确类型：

```typescript
// 字符串字面量
type Status = 'pending' | 'active' | 'closed'

// 数字字面量
type HTTPCode = 200 | 404 | 500

// 布尔字面量
type Truthy = true

// 使用
let status: Status = 'pending'
// status = 'unknown'  // ❌ 只能是定义的值之一

// as const 让值成为字面量类型
const task = { title: '学习 TS' } as const
// task.title = 'xxx'  // ❌ readonly
// typeof task.title => '学习 TS'（不是 string）
```

## 实战：完善 Task 数据模型

为 Task Manager 定义更完善的类型：

```typescript
// 任务状态
type TaskStatus = 'todo' | 'in-progress' | 'done'

// 任务优先级
type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

// 任务数据模型
interface Task {
  readonly id: number
  title: string
  description: string
  status: TaskStatus
  priority: TaskPriority
  tags: readonly string[]
  createdAt: Date
}

// 创建任务
function createTask(
  id: number,
  title: string,
  priority: TaskPriority = 'medium',
): Task {
  return {
    id,
    title,
    description: '',
    status: 'todo',
    priority,
    tags: [],
    createdAt: new Date(),
  }
}

// 更新状态
function updateStatus(task: Task, status: TaskStatus): Task {
  return { ...task, status }
}

// 打印任务
function printTask(task: Task): void {
  const icon = task.status === 'done' ? '✓' : task.status === 'in-progress' ? '◐' : '○'
  console.log(`[${icon}] #${task.id} ${task.title} (${task.priority})`)
}

// 测试
const task = createTask(1, '学习 TypeScript', 'high')
const updated = updateStatus(task, 'in-progress')
printTask(updated)
// [◐] #1 学习 TypeScript (high)
```

## 类型兼容性

TypeScript 使用**结构化类型**（Structural Typing）——只看形状，不看名字：

```typescript
interface Point2D { x: number; y: number }
interface Point3D { x: number; y: number; z: number }

const p3: Point3D = { x: 1, y: 2, z: 3 }
const p2: Point2D = p3  // ✅ Point3D 包含 Point2D 的所有属性
```

多的可以赋给少的，但少的不能赋给多的：

```typescript
const p2: Point2D = { x: 1, y: 2 }
// const p3: Point3D = p2  // ❌ 缺少 z 属性
```
