# 类型体操与工具类型

> 上一章我们学习了装饰器。本章深入 TypeScript 类型系统的高级玩法——内置工具类型、映射类型、条件类型分发、模板字面量类型，用"类型编程"构建类型安全的事件系统。

TypeScript 的类型系统本身就是一门函数式编程语言——条件类型是 `if`，映射类型是 `map`，模板字面量是字符串操作。掌握这些，你就能写出"类型比实现还复杂"的高级代码。

## 内置工具类型

TypeScript 内置了大量工具类型，分为几大类：

### 属性修饰

```typescript
interface Task {
  id: number
  title: string
  description: string
  status: string
  createdAt: Date
}

// Partial — 所有属性变可选
type PartialTask = Partial<Task>
// { id?: number; title?: string; ... }

// Required — 所有属性变必选
type RequiredTask = Required<PartialTask>

// Readonly — 所有属性变只读
type ReadonlyTask = Readonly<Task>

// Mutable — 所有属性变可写（TS 5.x 可以用 -readonly）
type MutableTask = { -readonly [K in keyof Task]: Task[K] }
```

### 类型挑选

```typescript
// Pick — 选取部分属性
type TaskPreview = Pick<Task, 'id' | 'title' | 'status'>
// { id: number; title: string; status: string }

// Omit — 排除部分属性
type TaskCreate = Omit<Task, 'id' | 'createdAt'>
// { title: string; description: string; status: string }

// Extract — 从联合类型中提取
type StringOrNumber = string | number | boolean
type OnlyString = Extract<StringOrNumber, string>  // string

// Exclude — 从联合类型中排除
type NotBoolean = Exclude<StringOrNumber, boolean>  // string | number

// NonNullable — 排除 null 和 undefined
type MaybeString = string | null | undefined
type DefString = NonNullable<MaybeString>  // string
```

### 函数类型

```typescript
type Fn = (a: string, b: number) => boolean

// Parameters — 获取参数类型元组
type Params = Parameters<Fn>          // [a: string, b: number]
type FirstParam = Params[0]           // string

// ReturnType — 获取返回值类型
type Result = ReturnType<Fn>          // boolean

// ConstructorParameters — 构造函数参数
class Task { constructor(public title: string, public priority: string) {} }
type TaskCtorParams = ConstructorParameters<typeof Task>  // [title: string, priority: string]

// InstanceType — 获取实例类型
type TaskInstance = InstanceType<typeof Task>  // Task
```

### 其他

```typescript
// Record — 构造键值对类型
type TaskMap = Record<number, Task>
// 等价于 { [key: number]: Task }

// Awaited — 递归解开 Promise
type AsyncResult = Awaited<Promise<Promise<string>>>  // string

// NoInfer — 阻止类型推断（TS 5.4+）
function createTask<T>(title: NoInfer<T>, priority: T): void {}
createTask('学习', 'high')  // T 推断为 'high'，不推断 title
```

## 映射类型

映射类型是"类型的 map"——遍历键，变换值：

### 基本映射

```typescript
// 把所有属性变成 string
type Stringify<T> = {
  [K in keyof T]: string
}

type StringTask = Stringify<Task>
// { id: string; title: string; ... }
```

### 键的重映射（TS 4.1+）

```typescript
// 添加前缀
type Prefix<T, P extends string> = {
  [K in keyof T as `${P}${Capitalize<string & K>}`]: T[K]
}

type PrefixedTask = Prefix<Task, 'task'>
// { taskId: number; taskTitle: string; ... }
```

### 过滤属性

```typescript
// 只保留函数类型的属性
type FunctionProperties<T> = {
  [K in keyof T as T[K] extends Function ? K : never]: T[K]
}

class TaskManager {
  add(title: string): Task { return {} as Task }
  remove(id: number): boolean { return true }
  count: number = 0
}

type TaskManagerMethods = FunctionProperties<TaskManager>
// { add: (title: string) => Task; remove: (id: number) => boolean }
```

### 修饰符映射

```typescript
// 添加 readonly
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K]
}

// 移除 readonly
type Mutable<T> = {
  -readonly [K in keyof T]: T[K]
}

// 添加可选
type Optional<T> = {
  [K in keyof T]?: T[K]
}

// 移除可选
type AllRequired<T> = {
  [K in keyof T]-?: T[K]
}
```

## 条件类型分发

当条件类型的输入是联合类型时，会自动分发：

```typescript
type ToArray<T> = T extends any ? T[] : never

type Result = ToArray<string | number>
// 分发: ToArray<string> | ToArray<number>
// 结果: string[] | number[]
```

### 阻止分发

用 `[T]` 包裹：

```typescript
type NoDistribute<T> = [T] extends [any] ? T[] : never

type Result2 = NoDistribute<string | number>
// 结果: (string | number)[]
```

### 实用分发模式

```typescript
// 递归展开嵌套数组
type Flatten<T> = T extends Array<infer U> ? Flatten<U> : T

type Flat = Flatten<(string | number)[][]>  // string | number

// 提取对象值的所有类型
type ValueOf<T> = T[keyof T]

type Config = { host: string; port: number; debug: boolean }
type ConfigValues = ValueOf<Config>  // string | number | boolean
```

## 模板字面量类型

TS 4.1+ 支持在类型层面操作字符串：

### 基本用法

```typescript
type EventName = `on${'Click' | 'Hover' | 'Focus'}`
// 'onClick' | 'onHover' | 'onFocus'

type CSSProperty = `margin-${'top' | 'right' | 'bottom' | 'left'}`
// 'margin-top' | 'margin-right' | 'margin-bottom' | 'margin-left'
```

### 内置字符串工具类型

```typescript
type A = Uppercase<'hello'>      // 'HELLO'
type B = Lowercase<'HELLO'>      // 'hello'
type C = Capitalize<'hello'>     // 'Hello'
type D = Uncapitalize<'Hello'>   // 'hello'
```

### 结合映射类型

```typescript
// 自动生成 getter 类型
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
}

interface User {
  name: string
  age: number
}

type UserGetters = Getters<User>
// { getName: () => string; getAge: () => number }
```

### 解析路径参数

```typescript
type ExtractRouteParams<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? { [K in Param | keyof ExtractRouteParams<Rest>]: string }
    : T extends `${string}:${infer Param}`
      ? { [K in Param]: string }
      : {}

type Params = ExtractRouteParams<'/users/:userId/posts/:postId'>
// { userId: string; postId: string }
```

## 实战：类型安全的事件系统

用高级类型构建一个完全类型安全的事件发射器：

```typescript
// ===== 事件映射定义 =====

interface TaskEvents {
  taskCreated: { id: number; title: string }
  taskUpdated: { id: number; changes: Record<string, unknown> }
  taskDeleted: { id: number }
  taskStatusChanged: { id: number; from: string; to: string }
}

// ===== 类型工具 =====

// 事件名联合
type EventName<T> = keyof T & string

// 事件回调类型
type EventCallback<T, K extends EventName<T>> = (payload: T[K]) => void

// ===== 事件发射器 =====

class TypeSafeEventEmitter<Events extends Record<string, any>> {
  #listeners = new Map<EventName<Events>, Set<EventCallback<Events, any>>>()

  on<K extends EventName<Events>>(
    event: K,
    callback: EventCallback<Events, K>,
  ): () => void {
    if (!this.#listeners.has(event)) {
      this.#listeners.set(event, new Set())
    }
    this.#listeners.get(event)!.add(callback)

    // 返回取消订阅函数
    return () => this.off(event, callback)
  }

  off<K extends EventName<Events>>(
    event: K,
    callback: EventCallback<Events, K>,
  ): void {
    this.#listeners.get(event)?.delete(callback)
  }

  emit<K extends EventName<Events>>(
    event: K,
    payload: Events[K],
  ): void {
    this.#listeners.get(event)?.forEach(cb => cb(payload))
  }

  once<K extends EventName<Events>>(
    event: K,
    callback: EventCallback<Events, K>,
  ): () => void {
    const wrapper: EventCallback<Events, K> = (payload) => {
      callback(payload)
      this.off(event, wrapper)
    }
    return this.on(event, wrapper)
  }
}

// ===== 使用 =====

const emitter = new TypeSafeEventEmitter<TaskEvents>()

// ✅ 类型安全的订阅
emitter.on('taskCreated', ({ id, title }) => {
  console.log(`任务创建: #${id} ${title}`)
})

emitter.on('taskStatusChanged', ({ id, from, to }) => {
  console.log(`#${id}: ${from} → ${to}`)
})

// ❌ 类型错误
// emitter.on('taskCreated', ({ id, unknown }) => ...)   // 未知属性
// emitter.emit('taskCreated', { wrong: true })            // 缺少 id 和 title

// ✅ 正确触发
emitter.emit('taskCreated', { id: 1, title: '学习类型体操' })
emitter.emit('taskStatusChanged', { id: 1, from: 'todo', to: 'done' })

// 取消订阅
const unsubscribe = emitter.on('taskDeleted', ({ id }) => {
  console.log(`任务删除: #${id}`)
})
unsubscribe()  // 取消订阅
```

### 扩展：异步事件 + 错误处理

```typescript
class AsyncEventEmitter<Events extends Record<string, any>> {
  #listeners = new Map<EventName<Events>, Set<EventCallback<Events, any>>>()

  async emitAsync<K extends EventName<Events>>(
    event: K,
    payload: Events[K],
  ): Promise<void> {
    const callbacks = this.#listeners.get(event)
    if (!callbacks) return

    const promises = [...callbacks].map(async (cb) => {
      try {
        await cb(payload)
      } catch (error) {
        console.error(`事件处理器错误 [${event}]:`, error)
      }
    })

    await Promise.all(promises)
  }

  // ... on/off/once 同上
}
```
