# 环境搭建与第一个程序

> 欢迎来到 TypeScript 教程！本章从零开始，带你安装开发环境、配置编译器、运行第一个程序。这是整个教程的基础，后续所有章节都依赖本章搭建的环境。

## 安装 Node.js

TypeScript 的工具链依赖 Node.js，先确认你已安装：

```bash
node --version
# v20.x.x 或更高

npm --version
# 10.x.x 或更高
```

如果没有安装，访问 [nodejs.org](https://nodejs.org) 下载 LTS 版本。推荐使用 [nvm](https://github.com/nvm-sh/nvm)（macOS/Linux）或 [nvm-windows](https://github.com/coreybutler/nvm-windows) 管理多版本 Node。

## 安装 TypeScript

### 全局安装

```bash
npm install -g typescript
tsc --version
# Version 5.x.x
```

### 项目级安装（推荐）

大型项目推荐局部安装，避免团队成员版本不一致：

```bash
mkdir task-manager && cd task-manager
npm init -y
npm install typescript --save-dev
npx tsc --version
```

## tsconfig.json — 编译器配置

`tsc --init` 生成默认配置，但我们需要调整一些关键选项：

```bash
npx tsc --init
```

修改 `tsconfig.json`：

```json name="tsconfig.json"
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "declaration": true,
    "sourceMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

### 关键选项说明

| 选项 | 作用 | 推荐值 |
|------|------|--------|
| `target` | 编译输出的 JS 版本 | `ES2022`（支持现代语法） |
| `module` | 模块系统 | `NodeNext`（Node.js 项目） |
| `strict` | 开启所有严格检查 | `true`（必须开启） |
| `outDir` | 编译输出目录 | `./dist` |
| `rootDir` | 源码根目录 | `./src` |
| `declaration` | 生成 `.d.ts` 声明文件 | `true`（发布 npm 包时必须） |
| `sourceMap` | 生成 source map 方便调试 | `true` |

## ts-node — 直接运行 TS

每次修改都要手动编译太麻烦，`ts-node` 让你直接运行 `.ts` 文件：

```bash
npm install ts-node --save-dev
```

创建 `src/index.ts`：

```typescript name="src/index.ts"
const message: string = 'Hello, TypeScript!'
console.log(message)
```

直接运行：

```bash
npx ts-node src/index.ts
# Hello, TypeScript!
```

## 第一个程序：Hello Task Manager

现在创建我们的实战项目骨架：

### 项目结构

```
task-manager/
├── src/
│   └── index.ts
├── package.json
└── tsconfig.json
```

### 代码

先写一个最简单的版本，只用基础类型——后续章节会逐步引入更高级的类型特性：

```typescript name="src/index.ts"
// 用对象字面量类型定义任务（第3章会升级为 interface）
type Task = {
  id: number
  title: string
  done: boolean
  priority: string  // 后续章节会用更严格的类型替代
}

// 创建任务
function createTask(id: number, title: string, priority: string): Task {
  return { id, title, priority, done: false }
}

// 打印任务
function printTask(task: Task): void {
  const status = task.done ? '✓' : '○'
  console.log(`[${status}] #${task.id} ${task.title} (${task.priority})`)
}

// 主函数
function main(): void {
  const tasks: Task[] = [
    createTask(1, '学习 TypeScript', 'high'),
    createTask(2, '写代码', 'medium'),
    createTask(3, '喝咖啡', 'low'),
  ]

  console.log('=== Task Manager ===')
  tasks.forEach(printTask)
}

main()
```

> **注意**：`priority` 目前是 `string`，任何字符串都能传进去。第 2 章会学到联合类型，把它限制为 `'low' | 'medium' | 'high'`；第 3 章会把 `type Task` 升级为 `interface`。这就是 TypeScript 的渐进式增强思路——先用简单的类型跑起来，再逐步收紧。

### 运行

```bash
npx ts-node src/index.ts
```

输出：

```
=== Task Manager ===
[○] #1 学习 TypeScript (high)
[○] #2 写代码 (medium)
[○] #3 喝咖啡 (low)
```

## 编译与运行分离

生产环境通常先编译再运行 JS：

```bash
# 编译
npx tsc

# 运行编译产物
node dist/index.js
```

在 `package.json` 中添加快捷脚本：

```json name="package.json"
{
  "scripts": {
    "build": "tsc",
    "start": "ts-node src/index.ts",
    "dev": "node --watch --loader ts-node/esm src/index.ts"
  }
}
```

## 常见问题

### `Cannot find module 'xxx'`

安装对应的类型定义：

```bash
npm install @types/node --save-dev
```

### `ts-node` 报语法错误

确保 `tsconfig.json` 的 `module` 和 `moduleResolution` 设置正确。Node.js 项目用 `NodeNext`。

### 编译后找不到模块

检查 import 语句是否带了 `.js` 后缀（`NodeNext` 模式要求）：

```typescript
// NodeNext 模式
import { something } from './utils.js'  // 即使源文件是 .ts

// 非 NodeNext 模式
import { something } from './utils'     // 不带后缀
```
