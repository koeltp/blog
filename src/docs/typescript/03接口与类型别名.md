# 接口与类型别名

> 上一章我们用字面量类型收紧了 `TaskStatus`，但一个对象有七八个属性，逐个写类型注解很啰嗦。`interface` 和 `type` 就是解决这个问题的——它们让你给"数据形状"命名，一次定义到处复用。

在 TypeScript 中，几乎一切类型最终都在描述"数据的形状"。`interface` 和 `type` 是描述形状的两把利器。

## 接口 interface

### 基本用法

```typescript
interface User {
  id: number
  name: string
  email: string
}

const user: User = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com',
}
```

### 可选属性

属性名后加 `?` 表示可以不提供：

```typescript
interface Task {
  id: number
  title: string
  description?: string   // 可选
  dueDate?: Date         // 可选
}

const task: Task = { id: 1, title: '学习 TS' }  // ✅ 不需要 description
```

### 只读属性

`readonly` 修饰符让属性初始化后不可修改：

```typescript
interface Task {
  readonly id: number
  title: string
  readonly createdAt: Date
}

const task: Task = { id: 1, title: '学习', createdAt: new Date() }
task.title = '复习'      // ✅ 可以修改
// task.id = 2            // ❌ 只读
// task.createdAt = new Date()  // ❌ 只读
```

### 索引签名

当你不知道具体有哪些属性名，但知道值的类型时：

```typescript
// 任意字符串键，值为 number
interface ScoreMap {
  [key: string]: number
}

const scores: ScoreMap = {
  math: 95,
  english: 88,
  physics: 92,
}

// 数字索引（通常用于数组描述）
interface StringArray {
  [index: number]: string
}

const names: StringArray = ['Alice', 'Bob']
```

> 索引签名要求所有已知属性的类型必须是索引值类型的子类型：
> ```typescript
> interface Bad {
>   [key: string]: number
>   name: string  // ❌ string 不能赋给 number
> }
> ```

### 继承

```typescript
interface BaseEntity {
  id: number
}

interface AuditedEntity {
  createdBy: string
  updatedBy: string
}

// ✅ 这才是真正的“多继承”（多重扩展）
// Task 同时继承了 BaseEntity 和 AuditedEntity 两个接口
interface Task extends BaseEntity, AuditedEntity {
  title: string
  status: string
}

// 此时 Task 拥有：id, createdBy, updatedBy, title, status
```

### 声明合并

`interface` 的独特能力——同名接口会自动合并：

```typescript
interface Window {
  title: string
}

interface Window {
  width: number
  height: number
}

// 等价于
interface Window {
  title: string
  width: number
  height: number
}
```

这个特性常用于扩展第三方库的类型：

```typescript
// 扩展 Express 的 Request 类型
declare module 'express' {
  interface Request {
    userId?: string
  }
}
```

## 类型别名 type

### 基本用法

```typescript
type Point = {
  x: number
  y: number
}

const p: Point = { x: 1, y: 2 }
```

看起来和 `interface` 一样，但 `type` 能做的更多：

### 联合类型

```typescript
type Status = 'pending' | 'active' | 'closed'
type Result = Success | Error   // 联合两个对象类型
```

### 交叉类型

```typescript
type BaseEntity = { id: number; createdAt: Date }
type Task = BaseEntity & { title: string; status: Status }
```

### 原始类型别名

```typescript
type ID = number | string
type Callback = (data: unknown) => void
```

### 映射类型

```typescript
type Readonly<T> = { readonly [K in keyof T]: T[K] }
type Optional<T> = { [K in keyof T]?: T[K] }
```

## interface vs type — 怎么选？

| 特性 | interface | type |
|------|-----------|------|
| 描述对象形状 | ✅ | ✅ |
| 继承/扩展 | `extends` | `&` 交叉 |
| 声明合并 | ✅ 支持 | ❌ 不支持 |
| 联合类型 | ❌ | ✅ |
| 原始类型别名 | ❌ | ✅ |
| 映射/条件类型 | ❌ | ✅ |
| 计算属性 | ❌ | ✅ |

### 选择建议

- **描述对象形状** → 优先用 `interface`（可继承、可合并、错误信息更友好）
- **联合/交叉/映射/条件类型** → 只能用 `type`
- **扩展第三方类型** → 用 `interface`（声明合并）

```typescript
// ✅ 用 interface 描述对象
interface Task {
  id: number
  title: string
}

// ✅ 用 type 描述联合
type Status = 'todo' | 'in-progress' | 'done'

// ✅ 用 type 做工具类型
type PartialTask = Partial<Task>
```

## 函数类型

两种写法：

```typescript
// interface 写法
interface GreetFunction {
  (name: string): string
}

// type 写法（更常用）
type GreetFunction = (name: string) => string

// 使用
const greet: GreetFunction = (name) => `Hello, ${name}!`
```

### 可调用签名 + 属性

函数也可以有属性（jQuery 风格）：

```typescript
interface AjaxFunction {
  (url: string, options?: RequestInit): Promise<Response>
  get(url: string): Promise<Response>
  post(url: string, body: unknown): Promise<Response>
}
```

### 构造签名

用 `new` 描述可以 `new` 的函数：

```typescript
interface Constructor {
  new (name: string): { name: string }
}

function createInstance(CTOR: Constructor, name: string) {
  return new CTOR(name)
}
```

## 实战：定义 API 响应类型

为 Task Manager 定义前后端交互的类型契约：

```typescript
// ===== 通用类型 =====

// 分页请求
interface PageRequest {
  page: number
  pageSize: number
  keyword?: string
}

// 分页响应
interface PageResponse<T> {
  items: T[]
  total: number
  page: number
  pageSize: number
}

// API 响应包装
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

// ===== 业务类型 =====

type TaskStatus = 'todo' | 'in-progress' | 'done'
type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

interface Task {
  readonly id: number
  title: string
  description: string
  status: TaskStatus
  priority: TaskPriority
  tags: string[]
  createdAt: string
  updatedAt: string
}

// 创建任务请求（不含 id 和时间戳）
interface CreateTaskRequest {
  title: string
  description?: string
  priority?: TaskPriority
  tags?: string[]
}

// 更新任务请求（所有字段可选）
interface UpdateTaskRequest {
  title?: string
  description?: string
  status?: TaskStatus
  priority?: TaskPriority
  tags?: string[]
}

// ===== API 类型 =====

type TaskListResponse = ApiResponse<PageResponse<Task>>
type TaskDetailResponse = ApiResponse<Task>
type CreateTaskResponse = ApiResponse<Task>
type UpdateTaskResponse = ApiResponse<Task>
type DeleteTaskResponse = ApiResponse<null>

// ===== 使用示例 =====

function handleListResponse(response: TaskListResponse): void {
  const { items, total } = response.data
  console.log(`共 ${total} 条任务`)
  items.forEach(task => {
    console.log(`#${task.id} ${task.title}`)
  })
}
```

## typeof — 从值获取类型

```typescript
const defaultConfig = {
  host: 'localhost',
  port: 3000,
  verbose: false,
}

// 从值推导类型
type Config = typeof defaultConfig
// 等价于 { host: string; port: number; verbose: boolean }

// 配合 as const 获取字面量类型
const constants = {
  MAX_RETRY: 3,
  TIMEOUT: 5000,
} as const

type Constants = typeof constants
// { readonly MAX_RETRY: 3; readonly TIMEOUT: 5000 }
```

## 常见陷阱与调试技巧

### 陷阱：索引签名的类型冲突

```typescript
// ❌ 具名属性类型必须兼容索引签名的值类型
interface Config {
  [key: string]: string | number
  port: number        // ✅ number 是 string | number 的子类型
  version: string     // ✅
  debug: boolean      // ❌ boolean 不是 string | number 的子类型
}

// ✅ 方案一：把索引值类型放宽
interface Config {
  [key: string]: string | number | boolean
  port: number
  debug: boolean
}

// ✅ 方案二：不用索引签名，用精确类型
interface Config {
  port: number
  version: string
  debug: boolean
}
```

### 陷阱：interface 继承交叉类型的差异

```typescript
interface A { x: number }
interface B { x: string }

// ❌ interface extends 同名属性会报冲突错误
// interface C extends A, B {}  // Error: x 类型不兼容

// ✅ type 交叉会把同名属性变为 never
type C = A & B
const c: C = { } as C
// c.x 的类型为 never，无法使用
```

### 陷阱：readonly 只是编译时保护

```typescript
interface Task {
  readonly id: number
}

const task: Task = { id: 1 }
// task.id = 2  // ❌ 编译报错

// 但运行时可以绕过
(task as any).id = 2  // 运行时不报错，id 被修改

// ✅ 真正的运行时保护
const frozen = Object.freeze({ id: 1 })
frozen.id = 2  // TypeError (strict mode)
```

### 调试技巧：快速查看类型全貌

在 VS Code 中选中一个变量/类型，按 `F12`（跳转定义）可以查看完整类型声明。对于复杂的推断类型，可以用工具类型打印：

```typescript
type TaskKeys = keyof Task        // 联合所有属性名
type TaskValues = Task[keyof Task]  // 联合所有属性值类型
```
