# 模块与声明文件

> 上一章我们用可辨识联合实现了多类型任务调度。本章学习 TypeScript 的模块系统和声明文件——如何组织代码、使用第三方库的类型、为没有类型的库编写声明。

TypeScript 完全采用 ES Modules 规范，同时增加了 `.d.ts` 声明文件机制来描述 JavaScript 的类型。

## ES Modules

### 导出

```typescript
// ===== 具名导出 =====

// types.ts
export type TaskStatus = 'todo' | 'in-progress' | 'done'
export type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

export interface Task {
  id: number
  title: string
  status: TaskStatus
  priority: TaskPriority
}

// ===== 默认导出 =====

// TaskManager.ts
export default class TaskManager {
  // ...
}
```

### 导入

```typescript
// 具名导入
import { Task, TaskStatus } from './types.js'

// 默认导入
import TaskManager from './TaskManager.js'

// 命名空间导入
import * as Tasks from './types.js'

// 重命名
import { Task as ITask } from './types.js'

// 只导入类型（编译后会被移除）
import type { Task, TaskStatus } from './types.js'
```

> NodeNext 模式下，相对路径导入必须带 `.js` 后缀（即使源文件是 `.ts`）。

### 重导出

```typescript
// index.ts — 桶文件
export { Task, TaskStatus } from './types.js'
export { default as TaskManager } from './TaskManager.js'
export type { TaskRepository } from './repository.js'
```

### 重新导出并重命名

```typescript
export { Task as ITask } from './types.js'
export { TaskManager as default } from './TaskManager.js'
```

## 声明文件

### 什么是 .d.ts

声明文件描述 JavaScript 代码的类型，自身不产生任何运行时代码：

```typescript
// math.d.ts
declare function add(a: number, b: number): number
declare function subtract(a: number, b: number): number
declare const PI: number
```

### 全局声明

```typescript
// global.d.ts
declare global {
  interface Window {
    myApp: {
      version: string
      init(): void
    }
  }

  // 扩展 Process
  interface Process {
    env: {
      NODE_ENV: 'development' | 'production' | 'test'
      API_URL: string
    }
  }
}

export {}  // 让文件成为模块，避免污染全局
```

### 模块声明

为没有类型的 npm 包编写声明：

```typescript
// declarations/unknown-lib.d.ts
declare module 'unknown-lib' {
  export function doSomething(value: string): number
  export interface Config {
    timeout: number
    retries: number
  }
  export const defaultConfig: Config
}
```

使用：

```typescript
import { doSomething, defaultConfig } from 'unknown-lib'
const result = doSomething('hello')  // number
```

## @types 生态系统

大多数 npm 包的类型定义由 DefinitelyTyped 社区维护：

```bash
# 安装 Node.js 类型
npm install @types/node --save-dev

# 安装 Express 类型
npm install @types/express --save-dev

# 查看某个包是否有类型
npm info @types/lodash
```

### 类型解析顺序

TypeScript 按以下顺序查找类型：

1. 包自身的 `types`/`typings` 字段（自带类型的包）
2. `node_modules/@types/xxx`（社区维护的类型）
3. `paths`/`baseUrl` 配置的路径
4. `typeRoots` 配置的目录

### tsconfig.json 配置

```json
{
  "compilerOptions": {
    "typeRoots": ["./node_modules/@types", "./src/types"],
    "types": ["node", "jest"]
  }
}
```

`types` 字段限制只加载指定的 `@types` 包，不设则自动加载所有。

## 模块增强（Module Augmentation）

给已有模块添加新的类型，而不修改源码：

### 扩展第三方模块

```typescript
// 扩展 Express 的 Request
declare module 'express' {
  interface Request {
    userId?: string
    sessionId?: string
  }
}

// 使用
import { Request, Response } from 'express'

function handler(req: Request, res: Response) {
  console.log(req.userId)  // ✅ 有类型提示
}
```

### 扩展全局对象

```typescript
// 扩展 Array 原型
declare global {
  interface Array<T> {
    first(): T | undefined
    last(): T | undefined
  }
}

// 实现
if (!Array.prototype.first) {
  Array.prototype.first = function () {
    return this[0]
  }
}
```

> 模块增强是"安全地猴子补丁"——有类型检查的版本。

## 三斜线指令

在 `.d.ts` 文件中引用其他声明文件：

```typescript
/// <reference path="./base.d.ts" />
/// <reference types="node" />
/// <reference lib="es2022" />
```

> 现代项目很少使用三斜线指令，`tsconfig.json` 已经能处理大部分场景。

## 实战：拆分为多文件项目

将 Task Manager 拆分为模块化的多文件结构：

```
src/
├── types/
│   ├── task.ts        # 任务相关类型
│   ├── api.ts         # API 响应类型
│   └── index.ts       # 桶文件
├── core/
│   ├── Task.ts        # Task 类
│   ├── TaskManager.ts # TaskManager 类
│   ├── Repository.ts  # 通用 Repository
│   └── index.ts
├── commands/
│   ├── parser.ts      # 命令解析器
│   ├── handler.ts     # 命令处理器
│   └── index.ts
├── declarations/
│   └── env.d.ts       # 环境变量类型
└── index.ts           # 入口
```

### 类型文件

```typescript name="src/types/task.ts"
export type TaskStatus = 'todo' | 'in-progress' | 'done'
export type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'

export interface Task {
  readonly id: number
  title: string
  description: string
  status: TaskStatus
  priority: TaskPriority
  tags: readonly string[]
  readonly createdAt: Date
  updatedAt: Date
}

export interface CreateTaskDTO {
  title: string
  description?: string
  priority?: TaskPriority
  tags?: string[]
}

export interface UpdateTaskDTO {
  title?: string
  description?: string
  status?: TaskStatus
  priority?: TaskPriority
}
```

```typescript name="src/types/api.ts"
import type { Task } from './task.js'

export interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

export interface PageResponse<T> {
  items: T[]
  total: number
  page: number
  pageSize: number
}

export type TaskListResponse = ApiResponse<PageResponse<Task>>
export type TaskDetailResponse = ApiResponse<Task>
```

```typescript name="src/types/index.ts"
export type { TaskStatus, TaskPriority, Task, CreateTaskDTO, UpdateTaskDTO } from './task.js'
export type { ApiResponse, PageResponse, TaskListResponse, TaskDetailResponse } from './api.js'
```

### 环境变量声明

```typescript name="src/declarations/env.d.ts"
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'production' | 'test'
    DB_PATH: string
    LOG_LEVEL: 'debug' | 'info' | 'warn' | 'error'
  }
}
```

### 入口文件

```typescript name="src/index.ts"
import { TaskManager } from './core/index.js'
import { parseCommand, handleCommand } from './commands/index.js'
import type { Command } from './commands/parser.js'

const manager = new TaskManager()

function repl(input: string): void {
  const command = parseCommand(input)
  if (!command) return

  const result = handleCommand(command, manager)
  console.log(result)
}

// 简单测试
repl('add 学习模块系统')
repl('list')
```

## 常见问题

### `Cannot find module './xxx'`

检查文件路径和扩展名。NodeNext 模式必须带 `.js` 后缀。

### `Could not find declaration file for module 'xxx'`

```bash
npm install @types/xxx --save-dev
```

如果 `@types/xxx` 不存在，自己写一个声明文件：

```typescript
// src/declarations/xxx.d.ts
declare module 'xxx' {
  const xxx: any
  export default xxx
}
```

### `Type originates at this import but must not be used in a computed property`

用 `import type` 导入只用于类型标注的模块。
