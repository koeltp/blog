# TypeScript 教程

TypeScript 是 JavaScript 的超集，在 JS 的基础上增加了静态类型系统和最新的 ECMAScript 特性。它由微软开发并开源，已成为大型前端工程的事实标准——Angular、Vue 3、React、Node.js 生态都在全面拥抱 TS。

## 为什么要学 TypeScript？

- **编译期查错** — 类型错误在编译时就能发现，而不是上线后崩给用户看
- **智能提示** — IDE 补全、重构、跳转定义，开发效率翻倍
- **自文档化** — 类型即文档，代码即契约，团队协作不再靠嘴说
- **生态支持** — npm 上 90%+ 的流行库都提供了 TS 类型定义
- **渐进式采用** — 可以从 JS 项目中逐步引入，不需要一次重写

```

第02章 → 定义 Task 数据模型（基础类型、类型推断）
第03章 → 定义 API 响应类型（接口、类型别名）
第04章 → 实现命令解析器（函数、重载）
第05章 → 封装 TaskManager 类（面向对象）
第06章 → 通用 Repository（泛型）
第07章 → 多类型任务调度（联合类型、类型守卫）
第08章 → 拆分为多文件项目（模块、声明文件）
第09章 → 异步文件持久化（Promise、async/await）
第10章 → 日志与权限装饰器（装饰器）
第11章 → 类型安全的事件系统（工具类型、类型体操）
第12章 → 完整工程化实践（ESLint、monorepo、发布）
```

## 学习建议

1. **边学边练** — 每章的代码示例都可以在 [TypeScript Playground](https://www.typescriptlang.org/play) 中直接运行
2. **跟着做项目** — 每章有实战环节，跟着做才能真正掌握
3. **完成练习题** — 每章末尾有 3 道练习，检测学习效果
4. **避开常见陷阱** — 每章的"常见陷阱与调试技巧"总结了新手最常踩的坑，先看再写能少走弯路
5. **理解类型系统** — TS 的核心是类型系统，理解类型推断和窄化是关键
6. **善用工具类型** — 不要重复造轮子，TypeScript 内置了大量工具类型
