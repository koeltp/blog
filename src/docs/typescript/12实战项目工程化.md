# 实战：项目工程化

> 最后一章，我们把前面学到的所有知识整合起来，搭建一个完整的 TypeScript 工程化体系——代码规范、严格模式、monorepo 管理、npm 包发布。

写代码只是第一步，真正的工程化还需要：统一的代码风格、完善的类型检查、自动化的构建流程、可复用的包管理。

## ESLint 配置

### 安装

```bash
npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
npm install -D eslint-config-prettier eslint-plugin-prettier
```

### 配置文件

```javascript name=".eslintrc.cjs"
module.exports = {
  root: true,
  parser: '@typescript-eslint/parser',
  parserOptions: {
    ecmaVersion: 2022,
    sourceType: 'module',
    project: './tsconfig.json',
  },
  plugins: ['@typescript-eslint'],
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',
    'plugin:@typescript-eslint/recommended-requiring-type-checking',
    'prettier',
  ],
  rules: {
    // 类型相关
    '@typescript-eslint/no-explicit-any': 'error',
    '@typescript-eslint/no-unsafe-assignment': 'error',
    '@typescript-eslint/no-unsafe-call': 'error',
    '@typescript-eslint/no-unsafe-return': 'error',
    '@typescript-eslint/explicit-function-return-type': ['warn', {
      allowExpressions: true,
      allowTypedFunctionExpressions: true,
    }],

    // 代码质量
    '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
    '@typescript-eslint/consistent-type-imports': ['error', {
      prefer: 'type-imports',
    }],
    '@typescript-eslint/no-floating-promises': 'error',
    '@typescript-eslint/no-misused-promises': 'error',

    // 命名规范
    '@typescript-eslint/naming-convention': ['error', {
      selector: 'interface',
      format: ['PascalCase'],
      custom: { regex: '^I[A-Z]', match: true },
    }, {
      selector: 'typeAlias',
      format: ['PascalCase'],
    }],
  },
}
```

### 常用规则说明

| 规则 | 作用 |
|------|------|
| `no-explicit-any` | 禁止使用 `any` |
| `no-unsafe-*` | 禁止不安全的类型操作 |
| `no-floating-promises` | Promise 必须被 await 或 catch |
| `no-misused-promises` | 禁止把 Promise 传给不需要的地方 |
| `consistent-type-imports` | 统一用 `import type` 导入类型 |

## strict 模式实践

`strict: true` 开启所有严格检查，这是 TypeScript 项目的**底线配置**：

```json
{
  "compilerOptions": {
    "strict": true
  }
}
```

等价于同时开启：

| 选项 | 作用 |
|------|------|
| `strictNullChecks` | null/undefined 不能赋给其他类型 |
| `noImplicitAny` | 禁止隐式 any |
| `strictFunctionTypes` | 函数参数类型协变检查 |
| `strictBindCallApply` | bind/call/apply 参数检查 |
| `strictPropertyInitialization` | 类属性必须初始化 |
| `noImplicitThis` | 禁止隐式 any 的 this |
| `useUnknownInCatchVariables` | catch 的 error 类型为 unknown |

### 常见 strict 模式问题

**null 检查**

```typescript
// ❌ strictNullChecks 下报错
function getLength(text: string | null): number {
  return text.length  // Object is possibly 'null'
}

// ✅ 方式一：类型守卫
function getLength(text: string | null): number {
  if (text === null) return 0
  return text.length
}

// ✅ 方式二：可选链
function getLength(text: string | null): number {
  return text?.length ?? 0
}
```

**类属性初始化**

```typescript
// ❌ strictPropertyInitialization 下报错
class User {
  name: string  // Property 'name' has no initializer
}

// ✅ 方式一：构造函数赋值
class User {
  name: string
  constructor(name: string) {
    this.name = name
  }
}

// ✅ 方式二：明确赋值断言
class User {
  name!: string  // 告诉 TS 我会在其他地方初始化
}
```

**catch 错误类型**

```typescript
try {
  // ...
} catch (error) {
  // error 类型为 unknown（不是 any）
  // ❌ error.message
  // ✅
  if (error instanceof Error) {
    console.log(error.message)
  }
}
```

## monorepo 管理

用 npm workspaces 管理多个包：

### 项目结构

```
task-manager/
├── packages/
│   ├── core/          # 核心类型和逻辑
│   │   ├── src/
│   │   ├── package.json
│   │   └── tsconfig.json
│   ├── storage/       # 存储适配器
│   │   ├── src/
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── cli/           # 命令行界面
│       ├── src/
│       ├── package.json
│       └── tsconfig.json
├── package.json       # 根 package.json
└── tsconfig.base.json # 共享 TS 配置
```

### 根配置

```json name="package.json"
{
  "name": "task-manager",
  "private": true,
  "workspaces": ["packages/*"]
}
```

```json name="tsconfig.base.json"
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  }
}
```

### 子包配置

```json name="packages/core/tsconfig.json"
{
  "extends": "../../tsconfig.base.json",
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src"]
}
```

```json name="packages/core/package.json"
{
  "name": "@task-manager/core",
  "version": "1.0.0",
  "type": "module",
  "main": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js"
    }
  },
  "scripts": {
    "build": "tsc",
    "dev": "tsc --watch",
    "typecheck": "tsc --noEmit"
  }
}
```

### 子包引用

```json name="packages/storage/package.json"
{
  "name": "@task-manager/storage",
  "dependencies": {
    "@task-manager/core": "workspace:*"
  }
}
```

```typescript name="packages/storage/src/index.ts"
import type { Task } from '@task-manager/core'

export async function saveTasks(tasks: Task[]): Promise<void> {
  // ...
}
```

## npm 包发布

### 准备

1. 确保所有导出都有类型声明
2. 正确配置 `exports` 字段
3. 在 `.npmignore` 中排除源码

### package.json 关键字段

```json
{
  "name": "@your-scope/package-name",
  "version": "1.0.0",
  "description": "简短描述",
  "keywords": ["typescript", "task-manager"],
  "license": "MIT",
  "repository": { "type": "git", "url": "https://github.com/you/repo" },
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  },
  "files": ["dist"],
  "sideEffects": false,
  "peerDependencies": {},
  "devDependencies": {}
}
```

### 双格式构建（ESM + CJS）

用 `tsup` 一次构建两种格式：

```bash
npm install -D tsup
```

```typescript name="tsup.config.ts"
import { defineConfig } from 'tsup'

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['esm', 'cjs'],
  dts: true,
  clean: true,
  sourcemap: true,
})
```

### 发布流程

```bash
# 版本号
npm version patch  # 1.0.0 → 1.0.1
npm version minor  # 1.0.0 → 1.1.0
npm version major  # 1.0.0 → 2.0.0

# 构建
npm run build

# 发布
npm publish --access public

# scoped 包首次发布
npm publish --access public
```

### 变更日志

用 `changeset` 管理版本和变更日志：

```bash
npm install -D @changesets/cli
npx changeset init
```

```bash
# 记录变更
npx changeset

# 版本升级 + 生成 CHANGELOG
npx changeset version

# 发布
npx changeset publish
```
