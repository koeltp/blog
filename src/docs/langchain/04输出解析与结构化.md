## 一、为什么需要输出解析

LLM 默认输出纯文本，但应用需要结构化数据：

```
# LLM 原始输出（不可靠）
"订单号是 ORD001，状态是已发货，预计明天到达。"

# 我们需要的（结构化）
{"order_id": "ORD001", "status": "shipped", "eta": "明天"}
```

## 二、三种解析方式对比

| 方式 | 原理 | 可靠性 | 推荐度 |
|------|------|--------|--------|
| `with_structured_output` | function calling / tool_use | 最高 | 首选 |
| `PydanticOutputParser` | Prompt 注入格式 + 后解析 | 中等 | 备选 |
| `JsonOutputParser` | Prompt 注入格式 + JSON 解析 | 中等 | 简单场景 |

## 三、with_structured_output（首选）

直接让模型按 Pydantic 模型输出，底层用 function calling，最可靠：

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

# 定义输出结构
class OrderStatus(BaseModel):
    """订单状态查询结果"""
    order_id: str = Field(description="订单编号")
    status: str = Field(description="订单状态：pending/shipped/delivered/cancelled")
    estimated_delivery: str | None = Field(default=None, description="预计送达时间")
    tracking_number: str | None = Field(default=None, description="物流单号")

# 绑定结构化输出
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
structured_llm = llm.with_structured_output(OrderStatus)

# 调用
result = structured_llm.invoke("帮我查一下订单 ORD20240601 的状态")
print(result)
# OrderStatus(order_id='ORD20240601', status='shipped',
#             estimated_delivery='明天下午', tracking_number='SF1234567890')

# 直接当 Pydantic 对象用
print(result.order_id)    # ORD20240601
print(result.status)      # shipped
```

### 3.1 嵌套结构

```python
class ProductInfo(BaseModel):
    name: str = Field(description="商品名称")
    price: float = Field(description="价格")

class OrderDetail(BaseModel):
    order_id: str
    products: list[ProductInfo] = Field(description="商品列表")
    total: float = Field(description="总金额")

structured_llm = llm.with_structured_output(OrderDetail)
result = structured_llm.invoke("订单 ORD001 包含一件iPhone 15 Pro 8999元和一件手机壳 59元")
```

### 3.2 枚举约束

```python
from enum import Enum

class Intent(str, Enum):
    ORDER_QUERY = "order_query"       # 订单查询
    RETURN_REQUEST = "return_request" # 退货
    PRODUCT_INFO = "product_info"     # 商品咨询
    COMPLAINT = "complaint"           # 投诉
    OTHER = "other"                   # 其他

class CustomerIntent(BaseModel):
    intent: Intent = Field(description="用户意图分类")
    confidence: float = Field(ge=0, le=1, description="置信度 0-1")

structured_llm = llm.with_structured_output(CustomerIntent)
result = structured_llm.invoke("我的快递怎么还没到")
print(result.intent)       # Intent.ORDER_QUERY
print(result.confidence)   # 0.95
```

## 四、PydanticOutputParser

当模型不支持 function calling 时使用：

```python
from langchain_core.output_parsers import PydanticOutputParser

parser = PydanticOutputParser(pydantic_object=OrderStatus)

# 查看自动生成的格式说明
print(parser.get_format_instructions())
# 输出类似：
# "The output should be formatted as a JSON instance that conforms to..."

# 在 Prompt 中注入格式说明
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是客服助手。{format_instructions}"),
    ("human", "{question}"),
])

chain = prompt | llm | parser

result = chain.invoke({
    "question": "查询订单 ORD001 状态",
    "format_instructions": parser.get_format_instructions(),
})
# result 是 OrderStatus 对象
```

## 五、JsonOutputParser

最简单的 JSON 解析，不需要 Pydantic 模型：

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "提取用户信息为JSON。{format_instructions}"),
    ("human", "{text}"),
])

chain = prompt | llm | parser
result = chain.invoke({
    "text": "我叫张三，手机号13800138000，VIP3级",
    "format_instructions": parser.get_format_instructions(),
})
# result 是 dict
```

## 六、解析失败处理

LLM 输出不可控，解析可能失败：

```python
from langchain_core.output_parsers import PydanticOutputParser
from pydantic import ValidationError

parser = PydanticOutputParser(pydantic_object=OrderStatus)

try:
    result = parser.parse(llm_output)
except (ValidationError, ValueError) as e:
    # 方案1：重试
    # 方案2：返回默认值
    result = OrderStatus(order_id="unknown", status="unknown")
```

**with_structured_output 的优势**：它底层用 function calling，模型会严格按格式输出，几乎不会解析失败。

## 七、实战：客服意图识别

```python
from enum import Enum
from pydantic import BaseModel, Field
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

class Intent(str, Enum):
    ORDER_QUERY = "order_query"
    RETURN_REQUEST = "return_request"
    PRODUCT_INFO = "product_info"
    COMPLAINT = "complaint"
    OTHER = "other"

class ParsedIntent(BaseModel):
    intent: Intent = Field(description="用户意图")
    entities: dict[str, str] = Field(
        default_factory=dict,
        description="提取的实体，如 order_id、product_name 等"
    )
    response_hint: str = Field(description="建议的回复方向")

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
intent_chain = llm.with_structured_output(ParsedIntent)

# 测试
test_inputs = [
    "我的订单 ORD001 怎么还没到",
    "iPhone 15 Pro 有没有蓝色",
    "我要退货，订单号 ORD002，商品有质量问题",
]

for text in test_inputs:
    result = intent_chain.invoke(text)
    print(f"输入: {text}")
    print(f"意图: {result.intent}, 实体: {result.entities}")
    print(f"建议: {result.response_hint}\n")
```

## 八、小结

| 方式 | 适用场景 | 可靠性 |
|------|---------|--------|
| `with_structured_output` | 模型支持 function calling | 最高 |
| `PydanticOutputParser` | 模型不支持 function calling | 中等 |
| `JsonOutputParser` | 简单 JSON 提取 | 中等 |
