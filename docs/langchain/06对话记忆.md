## 一、为什么需要记忆

LLM 本身没有记忆——每次调用都是独立的。客服机器人必须记住上下文：

```
用户：我叫张三
AI：张三您好！
用户：我叫什么名字？
AI：？？？（没有记忆就不知道）
```

## 二、ChatMessageHistory

LangChain 用 `ChatMessageHistory` 管理对话历史：

```python
from langchain_community.chat_message_histories import ChatMessageHistory

history = ChatMessageHistory()
history.add_user_message("我叫张三")
history.add_ai_message("张三您好！")

# 获取历史
print(history.messages)
# [HumanMessage(content='我叫张三'), AIMessage(content='张三您好！')]
```

## 三、RunnableWithMessageHistory

将对话历史自动注入链中：

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables import RunnableWithMessageHistory
from langchain_core.output_parsers import StrOutputParser
from langchain_community.chat_message_histories import ChatMessageHistory

# 链定义
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是客服助手，回答简洁有礼。"),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{question}"),
])

chain = prompt | ChatOpenAI(model="gpt-4o-mini") | StrOutputParser()

# 会话存储（内存版）
store: dict[str, ChatMessageHistory] = {}

def get_session_history(session_id: str) -> ChatMessageHistory:
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

# 包装为有记忆的链
chain_with_history = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="question",
    history_messages_key="history",
)

# 使用
response1 = chain_with_history.invoke(
    {"question": "我叫张三"},
    config={"configurable": {"session_id": "user-001"}},
)
print(response1)  # 张三您好！

response2 = chain_with_history.invoke(
    {"question": "我叫什么名字？"},
    config={"configurable": {"session_id": "user-001"}},
)
print(response2)  # 您叫张三。
```

## 四、记忆策略

直接把所有历史塞给 LLM 有两个问题：token 浪费和超出上下文窗口。

### 4.1 窗口记忆（只保留最近 K 轮）

```python
from langchain_core.messages import trim_messages

def trim_history(messages, max_tokens=2000):
    """裁剪历史消息，保留最近的消息"""
    return trim_messages(
        messages,
        max_tokens=max_tokens,
        strategy="last",       # 保留最后的消息
        token_counter=len,     # 简化计数（生产环境用 tiktoken）
        include_system=True,   # 保留系统消息
        allow_partial=False,
    )
```

### 4.2 摘要记忆（用 LLM 压缩历史）

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

summarize_prompt = ChatPromptTemplate.from_messages([
    ("system", "请将以下对话历史压缩为一段简洁的摘要，保留关键信息（人名、订单号、问题等）。"),
    ("human", "{history}"),
])

summarize_chain = summarize_prompt | ChatOpenAI(model="gpt-4o-mini") | StrOutputParser()

# 当历史超过阈值时，生成摘要替换旧消息
async def manage_history(session_id: str, max_messages: int = 10):
    history = get_session_history(session_id)
    if len(history.messages) > max_messages:
        # 旧消息生成摘要
        old_messages = history.messages[:-max_messages]
        summary = await summarize_chain.ainvoke(
            {"history": "\n".join(f"{m.type}: {m.content}" for m in old_messages)}
        )
        # 清空旧消息，插入摘要
        history.clear()
        history.add_ai_message(f"[对话摘要] {summary}")
```

### 4.3 策略对比

| 策略 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 全量记忆 | 信息完整 | token 消耗大 | 短对话 |
| 窗口记忆 | 简单高效 | 丢失早期信息 | 一般对话 |
| 摘要记忆 | 保留关键信息 | 额外 LLM 调用 | 长对话 |

## 五、持久化存储

内存存储重启后丢失，生产环境需要持久化：

### 5.1 Redis 存储

```python
from langchain_community.chat_message_histories import RedisChatMessageHistory

def get_session_history(session_id: str) -> RedisChatMessageHistory:
    return RedisChatMessageHistory(
        session_id=session_id,
        url="redis://localhost:6379",
        ttl=3600 * 24,  # 24小时过期
    )
```

### 5.2 文件存储

```python
from langchain_community.chat_message_histories import FileChatMessageHistory

def get_session_history(session_id: str) -> FileChatMessageHistory:
    return FileChatMessageHistory(f"./data/sessions/{session_id}.json")
```

### 5.3 数据库存储

```python
from langchain_community.chat_message_histories import SQLChatMessageHistory

def get_session_history(session_id: str) -> SQLChatMessageHistory:
    return SQLChatMessageHistory(
        session_id=session_id,
        connection="sqlite:///./data/sessions.db",
    )
```

## 六、小结

| 组件 | 用途 |
|------|------|
| `ChatMessageHistory` | 基础消息存储 |
| `RunnableWithMessageHistory` | 自动注入历史到链 |
| 窗口记忆 | 保留最近 K 轮 |
| 摘要记忆 | LLM 压缩历史 |
| 持久化 | Redis/文件/数据库存储 |
