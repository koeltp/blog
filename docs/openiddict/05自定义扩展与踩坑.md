## 一、自定义实体

OpenIddict 默认提供了四个实体类，但生产环境中你通常需要扩展它们（比如给 Application 加字段）。

### 1.1 默认实体 vs 自定义实体

| 默认实体 | 对应表 | 用途 |
| --- | --- | --- |
| OpenIddictApplication | OpenIddictApplications | 客户端应用注册 |
| OpenIddictAuthorization | OpenIddictAuthorizations | 用户授权记录 |
| OpenIddictToken | OpenIddictTokens | 令牌记录 |
| OpenIddictScope | OpenIddictScopes | 作用域定义 |

### 1.2 创建自定义实体

```csharp name="自定义实体"
public class MyApplication : OpenIddictApplication
{
    public string? Description { get; set; }
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
}

public class MyAuthorization : OpenIddictAuthorization
{
    public string? ConsentText { get; set; }
}

public class MyToken : OpenIddictToken
{
    public string? DeviceInfo { get; set; }
}

public class MyScope : OpenIddictScope
{
    public string? IconUrl { get; set; }
}
```

### 1.3 注册自定义实体

在 `AddCore` 中替换默认实体：

```csharp name="注册自定义实体"
.AddCore(options =>
{
    options.UseEntityFrameworkCore()
        .UseDbContext<AppDbContext>()
        .ReplaceDefaultEntities<
            MyApplication,
            MyAuthorization,
            MyScope,
            MyToken>();
})
```

### 1.4 更新 DbContext

```csharp name="DbContext 更新"
public class AppDbContext : IdentityDbContext<ApplicationUser, IdentityRole<Guid>, Guid>
{
    public DbSet<MyApplication> MyApplications { get; set; }
    public DbSet<MyAuthorization> MyAuthorizations { get; set; }
    public DbSet<MyToken> MyTokens { get; set; }
    public DbSet<MyScope> MyScopes { get; set; }

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);
        builder.UseOpenIddict<MyApplication, MyAuthorization, MyScope, MyToken, Guid>();
    }
}
```

## 二、自定义 Store

OpenIddict 默认使用 EF Core 存储数据。如果你用其他数据库或想完全控制存储逻辑，可以自定义 Store。

### 2.1 Store 接口

| 接口 | 操作对象 |
| --- | --- |
| IOpenIddictApplicationStore | 客户端应用 |
| IOpenIddictAuthorizationStore | 用户授权 |
| IOpenIddictTokenStore | 令牌 |
| IOpenIddictScopeStore | 作用域 |

### 2.2 自定义 TokenStore 示例

以 Redis 存储令牌为例：

```csharp name="Redis TokenStore"
public class RedisTokenStore : IOpenIddictTokenStore<OpenIddictToken>
{
    private readonly IDatabase _redis;

    public RedisTokenStore(IConnectionMultiplexer redis)
    {
        _redis = redis.GetDatabase();
    }

    public async ValueTask CreateAsync(OpenIddictToken token, CancellationToken ct)
    {
        var key = $"token:{token.Id}";
        await _redis.HashSetAsync(key, new HashEntry[]
        {
            ("application_id", token.ApplicationId.ToString()),
            ("type", token.Type),
            ("payload", token.Payload ?? ""),
            ("status", token.Status),
            ("subject", token.Subject ?? "")
        });
        await _redis.KeyExpireAsync(key, TimeSpan.FromDays(30));
    }

    public async ValueTask<OpenIddictToken?> FindByIdAsync(string id, CancellationToken ct)
    {
        var key = $"token:{id}";
        var entries = await _redis.HashGetAllAsync(key);
        if (entries.Length == 0) return null;

        var dict = entries.ToDictionary(e => e.Name.ToString(), e => e.Value.ToString());
        return new OpenIddictToken
        {
            Id = id,
            ApplicationId = dict.GetValueOrDefault("application_id"),
            Type = dict.GetValueOrDefault("type"),
            Payload = dict.GetValueOrDefault("payload"),
            Status = dict.GetValueOrDefault("status"),
            Subject = dict.GetValueOrDefault("subject")
        };
    }

    // ... 实现其他接口方法
}
```

### 2.3 注册自定义 Store

```csharp name="注册 Store"
.AddCore(options =>
{
    // 替换 TokenStore，其他 Store 保持默认
    options.ReplaceTokenStore<RedisTokenStore>();
})
```

> **注意**：自定义 Store 需要实现完整的接口（约 20-30 个方法），即使你只用到其中几个。建议参考 EF Core Store 的源码。

## 三、与 ASP.NET Core Identity 完整集成

### 3.1 完整配置

```csharp name="完整集成配置"
// 1. Identity
builder.Services.AddIdentity<ApplicationUser, IdentityRole<Guid>>(options =>
{
    options.Password.RequireDigit = true;
    options.Password.RequiredLength = 8;
    options.User.RequireUniqueEmail = true;
})
.AddEntityFrameworkStores<AppDbContext>()
.AddDefaultTokenProviders();

// 2. Cookie 认证（Identity 登录态）
builder.Services.AddAuthentication(options =>
{
    options.DefaultScheme = IdentityConstants.ApplicationScheme;
    options.DefaultChallengeScheme = IdentityConstants.ApplicationScheme;
})
.AddCookie(IdentityConstants.ApplicationScheme, options =>
{
    options.LoginPath = "/login";
    options.AccessDeniedPath = "/access-denied";
    options.ExpireTimeSpan = TimeSpan.FromMinutes(60);
});

// 3. OpenIddict
builder.Services.AddOpenIddict()
    .AddCore(/*...*/)
    .AddServer(/*...*/)
    .AddValidation(/*...*/);
```

### 3.2 登录页面

```csharp name="登录页面"
[HttpPost("/login")]
public async Task<IActionResult> Login(LoginViewModel model, string returnUrl)
{
    var user = await _userManager.FindByEmailAsync(model.Email);
    if (user == null || !await _userManager.CheckPasswordAsync(user, model.Password))
        return View(model with { Error = "用户名或密码错误" });

    if (!user.IsActive)
        return View(model with { Error = "账号已被禁用" });

    // 登录成功，设置 Identity Cookie
    await _signInManager.SignInAsync(user, isPersistent: model.RememberMe);

    // 跳转回 OpenIddict 授权流程
    return Redirect(returnUrl);
}
```

### 3.3 注销端点

```csharp name="注销端点"
[HttpGet("~/connect/logout")]
public IActionResult Logout() => View();

[HttpPost("~/connect/logout")]
[IgnoreAntiforgeryToken]
public async Task<IActionResult> LogoutPost()
{
    // 1. 注销 Identity 登录
    await _signInManager.SignOutAsync();

    // 2. 返回 OpenIddict 注销响应
    return SignOut(
        authenticationSchemes: OpenIddictServerAspNetCoreDefaults.AuthenticationScheme,
        properties: new AuthenticationProperties
        {
            RedirectUri = "/"
        });
}
```

## 四、常见踩坑问题

### 4.1 401 Unauthorized 错误

**症状**：API 请求带了正确的 Bearer Token，却返回 401。

**原因**：API 没有配置 JWT Bearer 认证中间件，或者 `AuthenticationSchemes` 不匹配。

```csharp name="修复 401"
// 确保注册了 JWT Bearer
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(/*...*/);

// API 上明确指定认证方案
[Authorize(AuthenticationSchemes = JwtBearerDefaults.AuthenticationScheme)]
```

### 4.2 跨域问题（CORS）

**症状**：浏览器报错 `No 'Access-Control-Allow-Origin' header`。

**原因**：认证服务器和资源服务器没配 CORS。

```csharp name="修复 CORS"
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowWeb", policy =>
        policy.WithOrigins("https://app.myapp.com", "http://localhost:5001")
              .AllowAnyHeader()
              .AllowAnyMethod()
              .AllowCredentials()); // 带 Cookie 时需要
});

// 中间件顺序：CORS → Authentication → Authorization
app.UseCors("AllowWeb");
app.UseAuthentication();
app.UseAuthorization();
```

### 4.3 令牌无法刷新

**症状**：`grant_type=refresh_token` 返回 `invalid_grant`。

**可能原因**：
1. Refresh Token 已被使用（Rotation 机制，用过的不能再用来）
2. Refresh Token 已过期（超过 14 天）
3. 用户被禁用（令牌端点检查了 `IsActive`）
4. `client_id` 不匹配（必须和原始签发时一致）

**调试方法**：
```csharp name="调试刷新令牌"
// 在令牌端点加日志
var result = await HttpContext.AuthenticateAsync(/*...*/);
if (!result.Succeeded)
{
    _logger.LogWarning("Refresh Token 验证失败: {Error}", result.Failure?.Message);
    return Forbid(/*...*/);
}
```

### 4.4 开发证书报错

**症状**：`Unable to resolve service for type 'X509Certificate2'` 或证书过期。

**原因**：`AddDevelopmentSigningCertificate()` 生成的证书有效期只有 2 年，且重启后会重新生成（导致之前签发的令牌无法验证）。

```csharp name="修复开发证书"
// 使用持久化的开发证书
options.AddDevelopmentSigningCertificate(new X509Certificate2("dev-sign.pfx"));
options.AddDevelopmentEncryptionCertificate(new X509Certificate2("dev-encrypt.pfx"));

// 或者用 openssl 生成
// openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes
// openssl pkcs12 -export -out cert.pfx -inkey key.pem -in cert.pem
```

### 4.5 声明丢失

**症状**：Access Token 里只有 `sub`，没有 `email`、`name` 等声明。

**原因**：OpenIddict 默认只包含 `sub`，其他声明需要通过 scope 控制。

```csharp name="修复声明丢失"
// 1. 确保注册了声明
options.RegisterClaims("name", "email", "role");

// 2. 确保在 ClaimsPrincipal 中添加了声明
identity.SetClaim(ClaimTypes.Name, user.DisplayName);
identity.SetClaim(ClaimTypes.Email, user.Email);

// 3. 确保设置了 scope
identity.SetScopes(request.GetScopes());

// 4. 确保 scope 映射到了声明
if (result.Principal.HasScope("email"))
    identity.SetClaim("email", user.Email);
```

## 五、调试技巧

### 5.1 启用 OpenIddict 日志

```csharp name="日志配置"
// appsettings.Development.json
{
    "Logging": {
        "LogLevel": {
```

### 5.2 解码 JWT

用 [jwt.io](https://jwt.io) 粘贴 Access Token 查看内容。如果看到 `sub`、`scope` 等字段，说明签发正常。

- **curl**：快速测试令牌端点
- **OpenIddict 控制台**：直接查 `OpenIddictTokens` 表看令牌状态

五篇教程覆盖了 OpenIddict 的核心使用：

1. **概述与快速上手**：概念、安装、服务注册、用户模型、种子数据
2. **授权码流程**：完整授权码+PKCE流程，三个端点的实现
3. **客户端凭证与资源服务器**：服务间通信，API 保护配置
4. **令牌管理**：刷新、撤销、引用令牌、安全实践
5. **自定义扩展与踩坑**：自定义实体/Store、Identity 集成、常见问题

**核心思想回顾**：
- OpenIddict 只负责协议层，不管 UI 和用户体验
- 令牌短命+刷新令牌=安全与便捷的平衡
- PKCE 和 Rotation 是现代 OAuth2 的安全底线
- 生产环境必须用真实证书，配置 HTTPS
